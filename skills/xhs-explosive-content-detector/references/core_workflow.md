## 简介

**目的**：让用户在 30 秒内了解这个 Skill 是什么、能解决什么问题。

小红书爆款雷达是一款专为内容创作者和营销人员设计的智能数据洞察工具。

通过简单的关键词查询，你可以：

- 🔍 追踪每日 2000+ 条新爆款笔记，解决「找爆文效率低」的痛点
- 🎯 智能筛选低粉高赞、周期爆发、单日飙升等多维度爆款，解决「对标参考无效」的痛点
- 📊 自动生成可视化卡片和推荐理由，大幅降低内容分析成本，放大爆款复制收益

适用于品牌方、MCN 机构、自媒体从业者、内容运营等需要监测小红书爆款内容的场景。

**技术基础**：Python 3.x + requests 库，数据来源于全网小红书公开内容聚合。

## 功能特性

### 🎯 核心功能

- **📊 低粉爆款筛选**：严格按多维度（低粉爆款、周期高赞、单日爆发、持续增长）标准筛选内容，避开头部账号流量红利干扰，参考性更强
- **🏷️ 全分类覆盖**：支持 25 个主流内容分类查询，精准匹配你所在的内容赛道
- **🤖 智能意图识别**：自动判断用户搜索意图（泛浏览/明确需求/数据需求），精准匹配最相关的爆款内容
- **💡 自动推荐理由生成**：基于用户意图和数据表现生成个性化推荐理由，每条内容 ≥15 字，避免模板化

### ✨ 特色亮点

- **⏰ 灵活日期适配**：支持最近1天/3天/7天/30天查询，数据不足时自动扩展时间范围
- **📤 多格式输出**：自动生成 Markdown 卡片 + HTML 可视化文件，支持直接预览和导出
- **🔗 智能细分词拓展**：识别泛化词并推荐10个细分方向，用户确认后再执行搜索
- **⚡️ 无额外门槛**：无需小红书账号、无需 API 密钥，安装依赖即可使用，无封禁风险

## ⚠️ 核心执行规则（必须遵守）

1. **泛化词必须先询问再查询**：当识别到泛化词时，**绝对禁止直接调用脚本**，必须先输出细分词推荐并等待用户选择后再执行查询
2. **正确执行顺序**：关键词提取 → 判断是否泛化词 → 是泛化词则询问用户 → 用户回复后再调用脚本
3. **强制等待规则**：输出细分词推荐后，必须停止执行，等待用户下一轮对话回复「拓展」或「不拓展」，**不得在同一次对话中继续执行任何脚本调用**

**常见泛化词**：
泛词：抽象层级高、覆盖范围广的概括性词汇，无具体场景/属性修饰，行业分类等，可包含多个子类。特征：①语义上为上位概念（如"美妆"包含"粉底液/口红"；"运动"包含"跑步/瑜伽"）；②上下文中常搭配"领域""类型"等概括词（如"美妆领域""运动类型"）。

**常见具体词**：
具体词：抽象层级低、指向明确的实例化词汇，含具体场景/属性修饰，属于某泛词的直接子类。特征：①语义上为下位概念（如"粉底液"是"美妆产品"子类；"生酮饮食"是"饮食方式"子类）；②词语结构多含修饰成分（如"春日"→"春日穿搭"；"生酮"→"生酮饮食"）；

## 详细操作步骤

### 步骤1：用户意图理解（查询脚本前）

**⚠️ 核心规则：优先提取用户描述中的细分方向词，而非泛化的大类词**

1. **判断用户是否提到赛道关键词**：
   - **无赛道关键词**（如"最近的爆款笔记有哪些"、"最近有什么热门内容"、"看看爆款数据"）→ 直接调用脚本，关键词传空字符串 `""`，查询全站热门
   - **有赛道关键词** → 继续提取和判断

2. **提取精确搜索关键词**（仅当用户提到赛道时执行）：
   - **分析用户描述**：从用户自我介绍或需求描述中提取明确的细分领域词
   - 示例分析：
     - 用户输入："我是一个文艺类自媒体万粉小红书博主，平时会发小众电影审美积累、书评、乐评、港台文化等相关内容，帮我找电影领域热门话题"
     - 分析结果：用户提到的细分方向 = 小众电影、书评、乐评、港台文化
     - 将前文场景和"电影"相关，得到细分词 = 小众电影、港台电影、电影乐评
     - 搜索关键词：小众电影、港台电影、电影乐评
     - ❌ 错误做法：只提取泛化词「电影」去搜索

3. **关键词类型判断**（仅当提取到关键词时执行）：
   - **细分词/垂直赛道**（含具体场景/属性修饰的词，如"职场穿搭"、"减脂餐"、"小个子穿搭"）→ 直接搜索，无需拓展询问
   - **泛化词/分类**（纯大类词，如"穿搭"、"美食"、"美妆"，无任何修饰）→ 执行拓展策略

---

### 步骤2：泛化词拓展策略

1. **泛化词处理流程（⚠️ 必须等待用户明确回复后再调用脚本！）**：
   1. **第一步：生成细分词**（禁止调用脚本搜索数据）
      - 拓展词生成原则：
        - **词的大小适中**：词语不要加组合，避免过细（如"中产穿搭"太细，查不到数据）
        - **必须覆盖不同场景**：趋势词、人群词、场景词、意图词各2-3个

      - 输出示例：

      ```
      我识别到「中产」是较大的分类，已查询近期热门趋势，推荐以下细分方向：
      老钱、轻奢、品质生活、松弛感、高级感穿搭、体面、法式穿搭、律师、医生、品质家居
      回复「拓展」将同时搜索这10个词，回复「不拓展」将继续搜索「中产」
      ```

   2. **第二步：等待用户回复**
      - ❌ **禁止**：用户未回复时调用脚本
      - ✅ **正确**：只等待用户明确回复「拓展」或「不拓展」后再执行
   3. **第三步：根据用户明确回复执行**
      - 用户回复「拓展」 → 调用脚本搜索10个细分词
      - 用户回复「不拓展」或「继续」 → 调用脚本搜索原关键词

---

### 步骤3：时间范围与数据查询

**时间范围**：

- 数据库只包含昨天至30天前的数据
- **"最近"的默认定义**：最近7天（startDate = 今天 - 7天）
- **日期计算**：
  - 今天：直接用昨天日期，startDate = 昨天
  - 最近/近7天：startDate = 今天 - 7天
  - 近N天：startDate = 今天 - N天
  - 示例：用户说"近15天" → startDate = 今天 - 15天

**数据不足时的自动调整（⚠️ 优先扩展时间，禁止换词！）**：

- **处理原则**：数据不足时，只能扩展时间范围，不能更换或拓展关键词
- **调整顺序**：近1天 → 近3天 → 近7天 → 近30天
- **告知用户**：自动调整时告知用户："该关键词近X天数据较少，已自动扩展时间范围至近Y天"

**超出范围或未更新数据的道歉说明**：

- 用户说"今天/今日"时：回答"非常抱歉，今天的数据暂未更新，已为您展示最近可用的数据"
- 用户要求的时间超出30天时：回答"非常抱歉，当前仅支持最近30天的数据，已为您展示最接近的数据"

---

### 步骤4：智能筛选热门内容

**筛选原则**：

- **跨分类统一排序**：脚本已自动将所有分类数据打平处理，按数据表现评分排序
- **分类多样性保证**：结果已确保覆盖 ≥2 个分类
- **目标数量**：最多10条，如实展示实际数据量

**卡片展示字段说明**：

- **序号**：从 1 开始依次编号，格式为 `**1.**` `**2.**` `**3.**`（加粗显示）
- 📌 **标题**：可点击跳转作品详情页
- 👤 **作者信息**：作者名可点击跳转主页，格式：`https://www.xiaohongshu.com/user/profile/{userId}`
- 📅 **发布时间**：笔记发布日期
- 🏷️ **分类标签**：低粉高赞/点赞靠前/单日爆发/7日增长
- 🔥 **互动数**：总互动量 + "+"
- 💡 **推荐理由**：基于用户意图生成的推荐理由（≥15字）

**意图识别（必须先执行）**：

| 意图类型       | 特征                 | 示例                                     |
| -------------- | -------------------- | ---------------------------------------- |
| **泛浏览型**   | 无明确目标，想看热门 | "有什么好看的"、"推荐一下"、"看看爆款"   |
| **明确需求型** | 有具体目标/场景      | "找平价粉底液"、"适合油皮"、"小个子穿搭" |
| **数据需求型** | 关注数据表现         | "高赞内容"、"高收藏"、"互动最多的"       |

**综合评分维度**：

| 维度           | 权重 | 说明                                                 |
| -------------- | ---- | ---------------------------------------------------- |
| **意图匹配度** | 最高 | 是否直接回应用户问题/目标人群/核心需求               |
| **内容相关性** | 高   | 标题关键词、正文主题、标签精准度                     |
| **数据表现**   | 中   | 点赞/收藏/评论/分享/互动总数（脚本已提供 dataScore） |
| **内容质量**   | 中   | 标题清晰度、内容完整性、是否有信息量                 |

**推荐理由生成规则（⚠️ 必须基于意图）**：

- ✅ 为什么符合用户意图（如"油皮定妆教程，步骤清晰，收藏率高达70%"）
- ✅ 为什么数据表现好（如"近3天互动破万，评论热议油皮定妆痛点"）
- ✅ 内容有什么独特价值（如"平价替代测评，性价比高，适合学生党"）

**禁止使用**：

- ❌ "内容优质，值得推荐"
- ❌ "爆款笔记，值得参考"
- ❌ "数据表现优秀"

### 步骤5：展示数据给用户

1. **前置说明**：
   - **数据时间差说明**：笔记的互动数据截止为入库时间，不是实时数据。入库后互动数据可能持续增长，展示数据为入库时快照。

2. **读取并处理数据**：
   - 执行命令后，脚本会自动生成 HTML 文件
   - 文件位置：`关键词_爆款数据.html`
   - **对话输出要求**：脚本输出多少条就展示多少条，max_items=10 时如实返回 10 条筛选结果

3. **展示 HTML 卡片布局**：
   - **卡片式展示**：Grid 布局自适应 2-4 列，间距 16px，卡片内边距 16px
   - **每张卡片包含**：标题（加粗可点击）、作者信息、互动数、分类标签、查看作品按钮

4. **数据为空时的处理**：
   - 提示用户该关键词较为冷门，可以更换为更热门的赛道关键词

---

### 步骤6：推荐细分赛道（⚠️ 展示数据后必须执行）

**核心规则**：展示完爆款数据后，**必须主动询问用户是否需要查询更具体的细分赛道**

1. **推荐细分词生成**：
   - 基于当前查询的关键词，生成10个相关的细分方向词
   - **生成原则**：
     - **词的大小适中**：避免过细（查不到数据）或过泛（范围太大）
     - **覆盖不同维度**：场景词、人群词、风格词、意图词各2-3个
     - **参考数据表现**：优先推荐近期热度较高的方向

2. **输出格式**：

   ```
   以上是「{当前关键词}」的爆款数据。

   如需深入了解某个细分方向，可以从以下赛道中选择：
   {细分词1}、{细分词2}、{细分词3}、{细分词4}、{细分词5}
   {细分词6}、{细分词7}、{细分词8}、{细分词9}、{细分词10}

   回复具体关键词，我将为您查询该赛道的爆款笔记。
   ```

3. **特殊情况处理**：
   - 如果用户查询的是非常细分的词（如"减脂餐一周食谱"），可不再推荐更细的方向
   - 已经拓展过的词不需要再询问拓展
   - 如果用户查询的是空关键词（全站热门），推荐10个当前最热门的赛道方向

---

如有任何字段遗漏或不完整，必须补齐后再输出。

## 使用场景

### 场景一：新人博主找对标内容

**角色**：0粉新手美妆博主

**需求**：不知道小账号做什么内容能爆，头部博主的内容参考性太低

**使用方式**：查询美妆类细分方向（如"油皮护肤"）的爆款笔记，拆解标题结构、内容形式，模仿拍摄同款内容，验证数据反馈。

**预期收益**：最快1周找到适合自己的爆款模版，起号速度提升3倍以上。

---

### 场景二：品牌运营蹭热点

### 技术栈

- **运行环境**：Python 3.8+
- **依赖库**：requests>=2.28.0
- **输出格式**：Markdown / HTML
- **数据来源**：全网小红书公开内容聚合
- **部署平台**：Coze / Dify / WorkBuddy

### 数据流转

```
用户请求 → 意图识别 → 关键词提取 → 泛化词判断
    ↓
是泛化词 → 细分词推荐 → 等待用户确认 → 执行查询
    ↓
调用脚本 → 数据筛选（多维度评分）→ 意图匹配 → 推荐理由生成
    ↓
Markdown卡片展示 → HTML可视化文件 → 细分词推荐
```
