# 小红书爆款雷达 / xhs-explosive-content-detector

---

## 简介

本 Skill 面向 **品牌种草、MCN 星探、自媒体创作与内容运营**：按赛道关键词与时间窗口**获取**小红书侧爆款笔记样本，支持多维度筛选与意图分层输出，并交付 **Markdown** 与 **HTML** 等形态，便于对标与选题复盘。

**核心价值**

- **规模化发现**：在收录维度内降低手工扒榜成本（规模与口径以工作流为准）。  
- **弱化头部偏差**：低粉爆款、周期高赞、单日爆发、持续增长等维度并举，提升可参考性。  
- **即拿即用**：结构化摘要 + 本地 HTML，便于内部分享与二次编辑。  

**适用对象**

品牌方、MCN、创作者与增长团队。执行判定、对话分支与脚本细节须遵循 **`references/core_workflow.md`**。

**运行环境**：**Python 3.8+**；**`requests>=2.28.0`**（见 `SKILL.md`）。数据来源于脚本对接的聚合接口，字段与过滤逻辑以工作流与脚本为准。

---

## 功能特性

### 核心功能

- **多维爆款筛选**：低粉爆款、周期高赞、单日爆发、持续增长等组合（以工作流为准）。  
- **赛道对齐**：支持约 **25** 个主流内容分类。  
- **意图分层**：区分泛浏览、明确需求与数据向诉求，匹配输出粒度与排序侧重。  
- **推荐理由**：结合意图与互动表现生成说明（单条字数等规则见工作流）。  

### 特色亮点

- **时间窗口弹性**：支持近 1 / 3 / 7 / 30 天；样本不足时按工作流扩展并明示。  
- **双形态输出**：Markdown 卡片 + 本地 **HTML**（默认文件名规则见 `SKILL.md` 与脚本）。  
- **细分词拓展**：识别泛化关键词后给出细分方向，**用户确认后再检索**。

## 前置条件

### 依赖说明

- **Python**：3.8 及以上。  
- **Python 包**：`requests>=2.28.0`。  

```bash
pip install "requests>=2.28.0"
```

### 环境变量配置

| 变量名 | 必填 | 说明 |
| --- | --- | --- |
| `XHS_API_PROXY` | 否 | 代理地址，示例：`http://127.0.0.1:7890` |
| `OUTPUT_PATH` | 否 | 生成的 `.html` 保存目录；默认脚本当前工作目录 |

**macOS（zsh）**

在 `~/.zshrc` 末尾添加（将值换成你的配置；不需要的变量可省略整行）：

```bash
export XHS_API_PROXY="http://127.0.0.1:7890"
export OUTPUT_PATH="/path/to/output/dir"
```

保存后执行：

```bash
source ~/.zshrc
```

**Windows（PowerShell）**

- **仅当前窗口有效**：执行后即生效，无需其他命令；关闭窗口后失效。

```powershell
$env:XHS_API_PROXY = "http://127.0.0.1:7890"
$env:OUTPUT_PATH = "D:\path\to\output\dir"
```

- **持久化到用户环境**：`setx` 后**当前窗口不会自动出现**该变量，需**关闭并重新打开**终端（或重启 IDE）后再用。

```powershell
setx XHS_API_PROXY "http://127.0.0.1:7890"
setx OUTPUT_PATH "D:\path\to\output\dir"
```

## 使用指南

### 你可以怎么说

| 你的说法 | 大致会得到什么 |
| --- | --- |
| 「近 7 天职场穿搭有什么低粉爆款」 | 确认关键词与日期 → 读工作流 → 执行脚本 → TOP 摘要与规律要点，并告知 HTML 路径 |
| 「查查小红书爆款」（信息不全） | 追问关键词或是否全站、日期与关注维度，避免擅自假定 |
| 「要看全站热门」 | 在参数约定下使用空关键词 + 指定起始日期 |

### 命令速查（在技能根目录执行）

```bash
python scripts/fetch_xhs_trends.py --keyword <关键词> --start-date <日期>
python scripts/fetch_xhs_trends.py --keyword "" --start-date <日期>
python scripts/fetch_xhs_trends.py --keyword "减脂餐,职场穿搭,健身" --start-date <日期>
```

### 使用提示

- 口头日期（如「昨天」「近一周」）须按 **`references/core_workflow.md`** 映射为脚本接受的日期格式。  
- **缺参数、依赖或网络失败**：应说明错误类型并给出重试或配置 `XHS_API_PROXY` 的建议。  
- **数据为空**：如实告知，**禁止**虚构标题、链接与互动数字。

## 使用场景

| 场景 | 角色 | 需求描述 | 使用方式 |
| --- | --- | --- | --- |
| 新号冷启动选题 | 自媒体创作者 | 需要对同级别账号可复制的爆文样本 | 关键词 + 近 7～30 天；优先看低粉维度 |
| 品牌节点种草 | 品牌小红书运营 | 活动前需要可评审的热点与形式参考 | 品类关键词检索；导出 HTML 做内审 |
| 签约与孵化前筛号 | MCN 内容星探 | 要高互动潜力轨迹而非单条偶然爆款 | 固定窗口多次抽样；关注持续增长等维度 |
| 竞标与周报 | 内容策划 / PM | 客户要「趋势 + 样本」一页纸结论 | 结构化摘要 + HTML；结论严格对应数据窗口 |

## 重要数据说明

- **数据真实性**：榜单与数字均来自脚本拉取结果；接口异常或无数据时须如实说明。  
- **合规与隐私**：遵守平台社区规范与本地法律；外链与正文片段遵循工作流中的脱敏与引用边界。  
- **能力边界**：推送订阅、PDF 导出等**仅在实际已实现时**承诺；默认交付以脚本 + HTML / Markdown 为准（见 `references/core_workflow.md`）。  

---

## 注意事项与限制

- **执行主体**：完整流程建议在**主 Agent** 执行，避免子 Agent 重复请求或绕过统一校验。  
- **禁止编造**：不得脱离脚本结果编造互动数据、排名与笔记元数据。  
- **代理与输出目录**：若网络受限，可按需配置 `XHS_API_PROXY`；若需固定 HTML 目录，可设置 `OUTPUT_PATH`。
