# 抖音账号搜索

## 📝 简介

输入关键词就能搜到抖音上的博主账号，账号名称、粉丝量、总获赞数、关注数、作品数一目了然，点一下链接就能跳转主页看详情。

## ✨ 功能特性

| 功能模块 | 能力描述 | 核心价值 |
|---------|---------|---------|
| 关键词搜索 | 输入关键词搜索抖音博主账号 | 精准发现目标赛道的账号 |
| 数据展示 | 展示账号名称、粉丝数、总获赞数、关注数、作品数 | 快速评估账号影响力 |
| 主页直达 | 每条结果附可点击的主页链接 | 一键跳转查看账号详情 |
| 翻页浏览 | 每页展示多条结果，支持多页翻看 | 不遗漏任何一个账号 |

## 🎯 适用对象

- 🔍 **内容创作者** — 找同赛道博主，研究对标账号的内容方向和运营思路。
- 📊 **运营 / 数据分析** — 快速摸清某个品类在抖音上有哪些头部账号，掌握竞争格局。
- 🏢 **品牌 / MCN** — 筛选潜在合作达人，按粉丝量评估账号价值，辅助签约决策。
- 🛒 **电商选品** — 搜索细分赛道的带货达人，找到匹配的推广人选。

## 🔄 工作流程

### Step 1：理解用户意图，提取关键词

**⚠️ 核心规则：语义理解优先，优先提取细分方向词，而非泛化大类词**

**1. 提取关键词**

- 从用户描述中提取 2~6 字的细分关键词
- **过滤冗余修饰词**：用户说的"xx达人""xx博主""xxKOL""xx网红""xx账号"，搜索关键词仅取核心部分 **xx**，去除末尾的"达人""博主""KOL""网红""账号"等类别修饰词。例如"美妆达人" → 搜索"美妆"，"健身博主" → 搜索"健身"
- 若用户意图模糊（如"帮我搜抖音博主"），主动询问：「请问你想搜索哪个方向或领域的抖音账号？」
- 不得在用户未提供关键词时擅自猜测并调用脚本

**2. 识别页码参数**（用户未指定时使用默认值）

- **页码**（`--page`）：默认 `1`
  - 用户提到"下一页"、"第2页" → 对应页码
  - 用户提到"上一页" → 当前页 - 1
  - 未提及页码 → `1`（默认）

### Step 2：调用搜索脚本

```bash
python3 ~/.agents/skills/douyin-account-search/scripts/search_douyin_account.py "<关键词>" [--page 页码]
```

**参数说明：**

| 参数     | 可选值     | 含义                         |
| -------- | ---------- | ---------------------------- |
| keyword  | 任意字符串 | 搜索关键词                    |
| `--page` | 正整数     | 页码，从1开始（默认1）        |

脚本以 JSON 格式输出：

| 字段       | 说明                           |
| ---------- | ------------------------------ |
| `accounts` | 匹配的账号列表（保持接口原始顺序）  |
| `page`     | 当前页码                       |
| `has_next` | 是否有下一页（true/false）      |
| `total`    | 当前页返回条数                  |

**⚠️ 完整数据读取规则：** 脚本输出 JSON 数据量较大时，终端可能截断中间内容并显示 `// This is the omitted part`。此时必须读取 Bash 输出中提示的完整文件路径（如 `Full output saved to: /path/to/file`），使用 Read 工具获取完整 JSON 后再生成表格，确保所有字段数据完整可见，不遗漏任何记录。

每条账号字段：

| 字段             | 说明          |
| ---------------- | ------------- |
| `nickname`       | 账号昵称      |
| `account_id`     | 抖音账号ID    |
| `follower_count` | 粉丝数        |
| `like_count`     | 总获赞数      |
| `follow_count`   | 关注数        |
| `video_count`    | 作品总数      |
| `signature`      | 账号简介      |
| `verified`       | 是否认证      |
| `profile_url`    | 主页链接      |

### Step 3：判断结果并展示

#### 情况 A：accounts 数量 > 0（有匹配结果）

**A1. 告知用户查询范围**

> 📊 关键词「**XXX**」查询到 **N 个**抖音账号（第 {page} 页），以下是详细数据：

**A2. 渲染 Markdown 表格（全部展示）**

```markdown
| #   | 账号名称         | 粉丝数  | 总获赞数 | 关注数 | 作品数 | 主页链接                    |
| --- | ---------------- | ------- | -------- | ------ | ------ | --------------------------- |
| 1   | 赵露思           | 305.2w  | 1.2亿    | 9      | 156    | [访问主页](profile_url)     |
```

**数字格式化规则：**
- `< 10000`：原始数字（如 `320`）
- `≥ 10000` 且 `< 1亿`：`x.xw` 格式（如 `1.2w`、`305.2w`）
- `≥ 1亿`：`x.x亿` 格式（如 `1.2亿`）

**昵称规则：** 完整展示，不截断

**主页链接规则：** 固定显示文字 `[访问主页](profile_url)`，点击可跳转至抖音账号主页

**A3. 翻页提示（⚠️ 紧接在表格之后，不可省略，每次必须输出）**

- 若 `has_next` 为 true：
> 📄 当前第 **{page}** 页。回复「下一页」继续查看。

- 若 `has_next` 为 false（已是最后一页）：
> 📄 当前第 **{page}** 页，已无更多数据。

**翻页时「下一页」调用规则：** 若用户回复「下一页」，调用脚本时递增页码：
```bash
python3 ~/.agents/skills/douyin-account-search/scripts/search_douyin_account.py "<关键词>" --page {page+1}
```

**⚠️ A1~A3 缺一不可，必须在同一轮输出中连续完成。若缺少 A3，视为执行错误，必须重新补充输出。**

#### 情况 B：accounts 数量 = 0（无匹配结果）

**B1. 抱歉提示**

> 😔 抱歉，未找到与「**XXX**」直接相关的抖音账号。

**⚠️ B1 必须在同一轮输出中完成，不得中断。**

### Step 4：结束输出

全部内容展示完毕后，不等待、立刻结束输出。
