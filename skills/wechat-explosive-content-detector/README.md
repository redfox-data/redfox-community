# 公众号爆款文章查询 / gzh-explosive-content-detector

---

## 简介

本 Skill 面向 **公众号选题与创作**：按关键词探测近期高热文章，结合「低粉高阅读、阅读靠前、数据增长中、原创靠前」等维度筛选，输出少量高质量推荐与 **HTML** 等形态，并给出可继续深挖的细分赛道词。

**核心价值**

- **关键词探测**：按赛道或细分词查询匹配爆款内容。  
- **智能筛选**：结合意图与相关性，从候选中挑出**最多 10 条**推荐。  
- **双形态展示**：Markdown 卡片 + HTML 网格卡片，便于阅读与分享。  
- **细分引导**：结果后附约 **10** 个可下一轮查询的细分词。  

**适用对象**

公众号运营与编辑、内容创作者、MCN 与增长团队。泛化词治理、脚本顺序与固定话术等须遵守 **`references/gzh_explosive_content_workflow.md`** 与 **`SKILL.md`**。

**运行环境**：**Python 3**；**`requests>=2.28.0`**（见 `SKILL.md` frontmatter）。

---

## 功能特性

### 核心功能

- **爆款文章探测**：支持全站热门（空关键词）与带关键词查询。  
- **时间窗口**：默认**最近 7 天**；最多约 **30 天**（以脚本与工作流为准）。  
- **泛化词治理**：大类词须先给细分推荐并等待「拓展 / 不拓展」，**禁止**同轮继续跑脚本。  
- **输出产物**：常见为 `{keyword}_爆款数据.html`；若脚本产出 JSON，可作为筛选参考。  

### 特色亮点

- **意图优先**：尽量用用户描述中的细分方向，避免只用过大类词一查了之。  
- **数据诚实**：非实时快照；对「今天」「超 30 天」等诉求使用**固定边界话术**（见「重要数据说明」）。  
- **禁止凑数**：最多 10 条，少于 10 条如实展示，**禁止编造**。  
- **展示闭环**：每条含推荐理由（字数与空话限制见 `SKILL.md`）；文末附细分词引导。

## 使用指南

### 你可以怎么说

| 你的说法 | 大致会得到什么 |
| --- | --- |
| 「最近有什么爆款」「看看全站热门」 | 关键词传空串查全站热门 → 卡片与 HTML → 再给细分赛道词 |
| 「职场 / 情感有什么爆款」（大类） | 先细分词列表并等待「拓展 / 不拓展」，**本轮不跑脚本** |
| 「职场沟通技巧」「亲子教育选题」 | 视为细分词，可直接进入脚本查询 |
| 「近 15 天的 xxx」 | 将口语映射为 `startDate = 今天 - N 天` 后查询 |

### 使用提示

- **泛化词必须先问再查**：输出细分推荐后须**停止本轮脚本**，等用户回复后再调用。  
- **关键词何时传空**：用户未给出任何赛道或主题、仅泛泛要看爆款时，传 `""` 查全站。  
- **数据为空**：提示更换更热门赛道词；**不得**用无关关键词硬凑或误触发泛化流程。

## 重要数据说明

**时间与实时性**

- 互动数据截止为**数据获取时刻**附近快照，后续可能持续增长。  
- 默认时间范围：**最近 7 天**；数据不足时可能**自动扩展**并按规则告知用户。  

**边界话术（与 `SKILL.md` 一致，执行时须原文使用）**

- 用户提及「今天 / 今日」：**「非常抱歉，今天的数据暂未更新，已为您展示最近可用的数据」**  
- 用户要求超出约 30 天：**「非常抱歉，当前仅支持最近30天的数据，已为您展示最接近的数据」**  

---

## 注意事项与限制

- **数据真实性**：展示内容须与脚本结果一致，**禁止编造**推荐理由与元数据。  
- **合规**：遵守平台与广告相关规范；细分推荐与正文引用注意边界。  
- **范围**：不承诺替代平台审核；发布前请自行复核。

## 常见问答（FAQ）

**Q：用户只说「职场」，能不能直接查？**  
A：**不能**。须先走泛化词拓展并等待「拓展 / 不拓展」，同一轮禁止脚本。

**Q：什么时候关键词传空？**  
A：用户**未给出任何赛道或主题词**、仅泛泛要看爆款 / 热门时，传 `""` 查全站。

**Q：数据为空怎么办？**  
A：提示关键词偏冷门，建议更换更热门赛道；**不得**改用无关关键词或误触发泛化拓展。
