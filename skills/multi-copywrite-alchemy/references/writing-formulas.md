# 写作公式参考手册

> 此文件列举所有支持的写作公式（文章结构模式），供"完美复刻蒸馏技能"在格式适配分析和写作模式选择中引用。

---

## 如何使用公式

蒸馏完成后，系统会根据作者风格推荐最适合的3个公式。用户也可以手动选择任意公式。

**格式塔原则**：公式提供结构骨架，但血肉（词汇、句式、语气、修辞）必须用蒸馏出的风格来填充。公式是"怎么搭"，风格是"用什么搭"。

## 一、核心叙事公式

### PAS — 问题 → 放大痛点 → 给解法

**适合**：发现了一个普遍存在的问题，想给解决方案。

**结构**：
1. **Problem**：一针见血指出问题
2. **Agitate**：放大痛点，让读者感到"说的就是我"
3. **Solve**：给出具体可操作的解决方案

**典型场景**：痛点文、方法论、工具推荐、课程推广
**字数建议**：1500-3000字
**判断标准**：文章核心是"教人做事"且问题已有清晰解法

### QUEST — 提问 → 共情 → 教学 → 激发 → 行动

**适合**：教程类、方法论类、需要建立信任的知识型内容。

**结构**：
1. **Question**：用一个读者心里正在想的问题开场
2. **Understand**：表达理解（"我当年也困惑过"）
3. **Educate**：系统性教学/讲解
4. **Stimulate**：激发——让读者意识到"原来可以这样"
5. **Transition**：引导行动

**典型场景**：教程文章、技能分享、知识科普
**字数建议**：2000-4000字
**判断标准**：文章核心是"教人做事"且需要先建立信任

### SCAR — 故事 → 冲突 → 顿悟 → 结果

**适合**：讲自己的真实经历，2026年最火的公式。

**结构**：
1. **Story**：一个具体的故事/场景
2. **Conflict**：故事中的冲突/困境/转折
3. **Aha**：顿悟时刻——"我突然明白了"
4. **Result**：结果和改变

**典型场景**：个人经历、成长故事、创业记录、情感故事
**字数建议**：1500-3500字
**判断标准**：文章核心是"讲个故事"

### AIDA — 注意 → 兴趣 → 欲望 → 行动

**适合**：产品介绍、工具推荐、转化型内容。

**结构**：
1. **Attention**：抓住注意力（一句话钩子）
2. **Interest**：引发兴趣（这是什么，为什么重要）
3. **Desire**：制造欲望（用了会怎样，不用会怎样）
4. **Action**：推动行动（购买/下载/关注）

**典型场景**：好物推荐、工具测评、课程推广
**字数建议**：1000-2500字
**判断标准**：文章核心目的是说服读者采取行动

## 二、产品与销售公式

### FAB — 特性 → 优势 → 利益

**适合**：产品介绍、测评、功能说明。

**结构**：
1. **Feature**：产品有什么特性
2. **Advantage**：这个特性相比竞品有什么优势
3. **Benefit**：对用户有什么实际利益

**典型场景**：3C测评、美妆种草、软件介绍
**字数建议**：800-2000字

### BAB — Before-After-Bridge

**适合**：对比类内容，先画饼再搭桥。

**结构**：
1. **Before**：现在的痛苦/不满
2. **After**：理想的状态
3. **Bridge**：如何从Before到After（你的产品/方法就是这座桥）

**典型场景**：课程销售、咨询服务、效果对比
**字数建议**：1000-2500字

### 4Ps — 承诺 → 想象 → 证明 → 推动

**适合**：卖课、卖服务的高客单价场景。

**结构**：
1. **Promise**：一个大承诺（"学完这门课你能…"）
2. **Picture**：描绘画面（"想象一下三个月后的你…"）
3. **Proof**：给出证明（案例、数据、背书）
4. **Push**：推一把（限时、限量、紧迫感）

**典型场景**：课程文案、高价服务介绍、社群招募
**字数建议**：2000-5000字

## 三、标题与钩子公式

### 4U's — 标题打磨公式

**结构**：
1. **Urgent**：紧迫感（限时、倒计时、错过即无）
2. **Unique**：独特性（别人没说过、独家发现）
3. **Useful**：实用性（看了就能用、用了就有效）
4. **Ultra-specific**：超具体（具体数字、具体方法、具体结果）

**用法**：写标题时至少满足3/4个U。

### 三大钩子模型

一篇文章选一个主钩子：

| 钩子类型 | 特点 | 用法 | 示例 |
|---------|------|------|------|
| **情绪钩子**（扎心） | 让读者感到"说的就是我" | 开篇第一句直接戳痛点 | "你以为是性格问题，其实是钱的问题。" |
| **观点钩子**（反常识） | 颠覆读者认知 | 开头抛出一个反直觉的观点 | "越穷的人越喜欢花钱买面子。" |
| **深度钩子**（秀肌肉） | 展示专业度 | 开头展示数据、研究或独特视角 | "我分析了3000条爆款标题，发现了一个反直觉的规律。" |

## 四、进阶公式

### 情绪曲线公式

**适合**：长文、销售页、需要深度共情的内容。

**结构**：
好奇 → 共鸣 → 焦虑 → 释然 → 行动

**要点**：
- 好奇（开头3秒内引发好奇）
- 共鸣（让读者觉得"你懂我"）
- 焦虑（放大不行动的后果）
- 释然（给出解决方案，情绪释放）
- 行动（在情绪高点推动转化）

**典型场景**：万字长文、品牌故事、深度销售页
**字数建议**：3000-8000字

### PDCA 内容迭代公式

**适合**：系列内容创作，把每篇的数据回流到下一篇。

**结构**：
1. **Plan**：基于上篇数据规划下篇内容
2. **Do**：按计划创作发布
3. **Check**：收集数据反馈（打开率、完读率、互动率）
4. **Act**：调整优化再出发

**用法**：作为内容创作者的工作流，不是单篇文章的结构。

### 三段式短内容

**适合**：短视频口播、15秒完播率最高的结构。

**结构**：
1. **Hook**（0-3秒）：一句话抓住注意力
2. **Body**（3-12秒）：核心内容/反转/高潮
3. **CTA**（12-15秒）：引导互动

### 列表体 2.0

**适合**：测评、推荐、对比类内容。

**结构**：
- 开头：设定标准（"测了X个，只有Y个值得用"）
- 主体：逐项测评，每个附带判断标准
- 结尾：总结推荐（按场景分推荐）

**典型场景**：工具测评、书籍推荐、资源汇总
**特征**：大量测试/体验后精选，有明确的淘汰标准

## 五、平台专用公式

### 爆款笔记四步结构（小红书验证）

**结构**：
1. **痛点即我**：开头就让读者觉得"这就是我的问题"
2. **情绪被接住**：表达理解和共情，不急着给解法
3. **可实践方法**：具体可操作的步骤（1/2/3…）
4. **收尾金句**：一句能让人截图收藏的话

### Hook-Stack-CTA 三明治（X / Thread 专用）

**结构**：
1. **Hook**：第一条推文的钩子（决定是否展开阅读）
2. **Stack**：中间的多条推文堆积信息和价值
3. **CTA**：最后一条推文的行动号召

**铁律**：每条推文独立可读、可转发，不依赖上下文。

## 六、纯文本硬核干货公式

### S.C.S — Situation → Complication → Solution

**适合**：顾问式分析、商业洞察、行业报告。

**结构**：
1. **Situation**：现状是什么
2. **Complication**：问题/挑战在哪里
3. **Solution**：怎么解决

### F.V.C — Fact → Value → Call-to-action

**适合**：资讯快报、数据解读、干货速递。

**结构**：
1. **Fact**：一个事实/数据
2. **Value**：为什么重要/意味着什么
3. **Call-to-action**：建议怎么应对
