# 小红书文案评分 / xhs-copywriting-score

---

## 简介

本 Skill 是 **小红书文案质量评分工具**：根据你提供的笔记正文，结合全网持续收录的爆款样本（技能说明为每日 **2000+** 条量级），从多个维度打分并给出可操作的改进方向，帮助提升文案结构与流量表现。

**核心价值**

- **四维评分**：关键词覆盖、结构完整度、时效性、内容质量，各维度满分 **100**，总分为四维平均（四舍五入取整）。  
- **数据驱动**：评分须基于真实爆款查询结果，**禁止**凭空打分。  
- **客观公正**：写得优秀不压分，能上 **90** 就上 **90**；**≥90** 为优秀，可直接发布导向。  
- **可选精修**：未达优秀时可给出改法指导，**只改低分维度对应段落**，保留你的语气与原味表达。  

**适用对象**

需要快速判断文案是否达标、想知道缺什么要素、希望在不大改风格前提下微调正文的博主与运营。

**运行环境**：**Python 3**；须安装 **`requests>=2.28.0`**。在已启用本 Skill 的 Agent 中粘贴文案或说明「帮我评分」即可触发。

---

## 功能特性

### 核心功能

- **接收正文**：从你提供的完整小红书文案中提取 **1～3** 个主题关键词，用于拉取同领域爆款参考。  
- **对标爆款**：在后台查询相关热门笔记规律（**不会**把原始爆款大表直接展示给你）。  
- **结构化输出**：总分与评级、维度拆解表、改进建议或亮点说明、参考爆款规律摘要、**2～3** 篇核心参考笔记链接与互动数据。  
- **优化模式（可选）**：你确认需要优化时，先输出改法指导（原句 / 改法 / 原因），再给出「优化后文案」。  

### 特色亮点

- **90 分优秀线**：优秀档祝贺可直接发布，**不再**追问是否优化。  
- **建议只动正文**：你仅提供正文时，改进建议**不**涉及标题优化。  
- **保护个人风格**：禁止整篇重写；评分 **≥70** 的维度相关内容尽量不改；口语化、网络用语优先保留。

## 使用指南

### 你可以怎么说

| 你的说法 | 大致会得到什么 |
| --- | --- |
| 「帮我看看这篇文案写得怎么样」+ 粘贴正文 | 四维评分 + 拆解 + 建议或亮点 + 参考笔记 |
| 「这篇文案有什么问题」「能火吗」 | 同上，侧重扣分说明与改进项 |
| 直接发送小红书正文 | 自动进入评分流程 |
| 「需要优化」/ 确认优化 | 改法指导 + 优化后完整正文（仅低分维度） |

### 评分维度速览

| 维度 | 关注点 |
| --- | --- |
| **关键词覆盖** | 是否包含领域高频热词 |
| **结构完整度** | 钩子、分点、干货、互动引导等 |
| **时效性** | 热点 / 季节 / 节日相关性 |
| **内容质量** | 干货密度、排版、Emoji、标签等 |

### 使用提示

- 输出开头会说明：评分基于真实爆款数据分析。  
- 若关键词与文案主题不匹配，会调整关键词重新查询后再评。  
- 你不会看到脚本命令、文件路径或原始爆款数据表。

## 使用场景

| 场景 | 适合谁 | 你能得到什么 |
| --- | --- | --- |
| **发布前自检** | 个人博主 | 是否达 90 分优秀线、缺哪些结构要素 |
| **改稿方向** | 运营 | 针对最低分维度的 2～3 条可执行建议 |
| **学习爆款结构** | 新人 | 参考规律公式 + 2～3 篇对标笔记 |
| **轻量润色** | 不愿大改风格的作者 | 仅改低于 70 分维度对应句子 |

---

## 注意事项与限制

- 评分依赖当次查询到的爆款数据，样本不足时结论可能偏保守。  
- **禁止**展示原始爆款表格；参考笔记须来自实际查询结果。  
- 优化时**禁止**整篇重写、禁止用「AI 味」替换你的原味表达。  
- 改法指导须具体可执行，避免「建议优化 XX」类空泛表述。  
- 遵守小红书平台内容规范。
