# Core Workflow

## Step 1：理解用户意图，提取 userId

**1. 提取小红书 userId**

- 从用户描述中提取小红书用户的 userId（如 `5a73c5fa4eacab4c4ccc9778`）
- 若用户提供了小红书主页链接，从 URL 中提取 userId，例如 `xiaohongshu.com/user/profile/5a73c5fa4eacab4c4ccc9778` 中的 `5a73c5fa4eacab4c4ccc9778`
- 若用户仅提供昵称而非 userId，告知用户：「小红书作品列表查询需要 userId，请在小红书用户主页 URL 中找到 userId 后告诉我，例如 xiaohongshu.com/user/profile/5a73c5fa4eacab4c4ccc9778 中的 5a73c5fa4eacab4c4ccc9778」
- 不得在用户未提供 userId 时擅自猜测并调用脚本

**2. 识别翻页参数**（用户未指定时使用默认值）

- **偏移量**（`--offset`）：默认不传（第一页）
  - 用户提到"下一页"、"更多" → 传上次返回的 offset 值
  - 首次查询 → 不传 offset

## Step 2：调用作品列表查询脚本

```bash
python ".qoder/skills/xhs-portfolio-search/scripts/fetch_xhs_user_works.py" "<userId>" [--offset 翻页偏移量]
```

**参数说明：**

| 参数       | 可选值       | 含义                                      |
| ---------- | ------------ | ----------------------------------------- |
| userId     | 字符串       | 小红书账号 userId                          |
| `--offset` | 偏移量字符串 | 翻页参数，第一页不传，传上次返回的 offset 值 |

脚本以 JSON 格式输出：

| 字段             | 说明                          |
| ---------------- | ----------------------------- |
| `user_info`      | 博主基础信息（昵称、头像等）   |
| `works`          | 作品列表                      |
| `offset`         | 下一页偏移量（空字符串表示无下一页） |
| `has_next`       | 是否有下一页（true/false）     |
| `total_count`    | 本页作品数量                  |

每条作品字段：

| 字段             | 说明                              |
| ---------------- | --------------------------------- |
| `title`          | 笔记标题                          |
| `author`         | 博主昵称                          |
| `thumb_count`    | 点赞数                            |
| `reply_count`    | 评论数                            |
| `favorite_count` | 收藏数                            |
| `forward_count`  | 分享数                            |
| `work_url`       | 笔记链接（直接可用）               |
| `release_time`   | 发布时间                          |
| `note_type`      | 类型（normal=图文, video=视频）    |
| `thumbnail`      | 封面图地址                        |
| `note_id`        | 笔记 ID                           |
| `pinned`         | 是否置顶                          |

## Step 3：判断结果并展示

### 情况 A：works 数量 > 0（有作品数据）

**A1. 展示博主信息 + 告知查询结果**

> 📊 小红书博主「**昵称**」（userId: XXX），当前展示 **N 条**作品，以下是详细数据：

**A2. 渲染 Markdown 表格（⚠️ 必须逐条全部展示，严禁截断）**

```markdown
| #   | 笔记标题               | 点赞数  | 收藏数  | 评论数 | 类型   | 发布时间   |
| --- | ---------------------- | ------- | ------- | ------ | ------ | ---------- |
| 1   | [标题文字](笔记链接)   | 28.8w   | 2.3w    | 3.4w   | 图文   | 2026-06-11 |
| 📌 | [置顶标题](笔记链接)   | 10.5w   | 1.1w    | 1.2w   | 视频   | 2026-06-01 |
```

**⚠️ 序号连续规则：**
- 第1页序号从1开始，第2页从21开始，第N页从 `(N-1)*20+1` 开始
- 置顶作品在表格中用 📌 标记，排在最前面
- 在上下文中记录当前页码，翻页时自动递增

**⚠️ 展示纪律（强制执行）：**
- 每页必须完整渲染 `works` 数组中的全部条目，严禁因篇幅原因截断或省略
- 标准页满20条，最后一页可不足20条，但有多少展示多少
- 表格行数必须与 `total_count` 字段一致，若不一致视为执行错误

**数字格式化规则：**
- `< 10000`：原始数字（如 `320`）
- `≥ 10000`：`x.xw` 格式（如 `1.2w`）

**标题规则：** `[标题](work_url)`；超过 30 字截断并加 `...`

**A3. 翻页提示（紧接在表格之后，每次必须输出）**

- 若本页 `works` 数量 == 20（满页）：
> 📄 还有更多作品。回复「下一页」继续查看。

- 若本页 `works` 数量 < 20（不满页）：
> **不展示任何翻页提示**

**A4. 订阅提示（紧接在 A3 之后输出）**

> 🔔 是否订阅「**{账号昵称}**」的作品追踪？支持以下推送频率：
> - 回复「**每日**」→ 每日 10:00 自动查询并推送最新作品
> - 回复「**每周**」→ 每周一 10:00 自动查询并推送最新作品
> - 回复「**自定义时间**」→ 告诉我你希望的推送时间，如「每周三 14:00」「每天 20:00」

**⚠️ A1~A4 缺一不可，必须在同一轮输出中连续完成。若缺少 A4 订阅提示，视为执行错误。**

### 情况 B：works 数量 = 0（无作品数据）

> 😔 抱歉，未查询到 userId「**XXX**」的作品数据。可能原因：1) 该账号暂无公开笔记 2) userId 不正确 3) 账号已注销。请核实 userId 后重试。

## Step 4：记录偏移量和页码（内部逻辑，不展示给用户）

将本次返回的 `offset` 值和当前**页码**记录在上下文中。当用户回复「下一页」时，使用该 offset 值再次调用脚本获取下一页数据，页码 +1。
- 首次查询：页码 = 1
- 每次翻页：页码 +1

## Step 5：处理订阅请求

当用户回复订阅意图时，使用 `schedule` MCP 工具创建定时任务。

**任务内容：** 调用作品列表查询脚本获取该用户最新作品并推送结果。

**调度规则：**

| 用户回复 | 调度配置 | 说明 |
|---------|---------|------|
| 「每日」 | 每天 10:00 | cron: `0 10 * * *` |
| 「每周」 | 每周一 10:00 | cron: `0 10 * * 1` |
| 自定义时间 | 根据用户描述解析 | 如「每周三 14:00」→ `0 14 * * 3` |

**任务 prompt 模板：**

```
查询小红书用户 {userId}（{昵称}）的最新作品列表，使用 xhs-portfolio-search 技能执行查询并展示结果。
```

**创建成功后回复：**

> ✅ 订阅成功！将在 **{频率描述}** 自动为你查询「**{昵称}**」的最新作品并推送。
> 如需取消订阅，回复「取消订阅」即可。
