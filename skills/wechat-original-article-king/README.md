# 公众号文章原创之王 / wechat-original-article-king

---

## 简介

本 Skill 面向 **公众号原创文章榜单查询**：按分类、日期或时间范围拉取**原创热门文章**列表，在对话中**原样展示**脚本输出的 Markdown 表格与说明，并自动生成可导出 **PDF** 的 **HTML** 榜单页；可在文末引导订阅赛道推送（是否已开通以产品侧为准）。

**核心价值**

- **数据来自脚本**：榜单须由 **`fetch_articles.py`** 拉取，**禁止**手写或编造文章、阅读数与链接。  
- **多维度查询**：支持 **总排名**、**22+ 标准分类**、指定日期；口语「最近 / 最新」默认 **近 7 天**。  
- **首轮 Top20**：常见展示前 **20** 条；总量更多时可提示是否继续查看。  
- **HTML + PDF**：微信绿版式页面，**`display_count`** 与对话展示条数一致，支持 **html2pdf.js** 单页导出。  

**适用对象**

公众号编辑与运营、内容策划、需要「原创爆文样本」做选题与对标的研究者或团队。

**运行环境**：**Python 3**（标准库即可）；数据脚本 **`scripts/fetch_articles.py`**，页面脚本 **`scripts/generate_hot_html.py`**。在已启用本 Skill 的 Agent 中通过自然对话触发。

---

## 功能特性

### 核心功能

- **意图识别**：区分总榜、分类、指定日期、「最近 / 最新」等说法，映射到脚本参数。  
- **stdout 即正文**：数据说明、表格、条数提示、订阅询问等须**完整原样**呈现，不得删改或替换成示例数据。  
- **自动 HTML**：每次输出表格后生成榜单 HTML（无需你再单独说「生成页面」）。  
- **只做榜单**：**不**自行写爆文规律分析、写作建议或二次创作。  

### 特色亮点

- **时间口径透明**：库表 **每日 19:30** 同步前一日数据；查询日期与用户口头「今天 / 昨天」不一致时，使用固定致歉说明。  
- **分类映射**：用户口语（如「科技类」）映射到标准分类名（如「科技数码」）再查询。  
- **冷门提示**：某赛道数据过少时，提示可换分类或扩大时间范围。

## 使用指南

### 你可以怎么说

| 你的说法 | 大致会得到什么 |
| --- | --- |
| 「最新原创爆文」「今日原创文章」「爆文推荐」 | 总榜 Top20 表格 + 数据说明 + 订阅问句 + HTML |
| 「科技 / 健康 / 财经…类原创爆款」 | 匹配分类后 Top20 + HTML |
| 「5 月 3 日的原创爆款」「昨日爆文」 | 指定日期查询 + 表格 + HTML |
| 「最近 / 最新有什么原创热门」 | 默认近 7 天范围 Top20 |

### 使用提示

- 对话中**不会**展示脚本路径或源码；你看到的是脚本运行结果。  
- 若只要 PDF / 页面，仍须先看到完整表格输出，再生成 HTML。  
- 加载本 Skill 后，可能会先问你是否需要推送最新原创爆款（可回复需要或跳过）。

## 重要数据说明

**同步与快照**

- 数据 **每日 19:30** 更新昨日口径；展示为**获取时刻快照**，与线上实时阅读可能不一致。  
- 未指明日期时，「最近 / 最新」按 **过去 7 天** 理解。  
- 当前时间 **早于 19:30** 与 **晚于 19:30**，「最新可查日期」策略不同（由脚本按规则计算）。  

**数量提示**

- 某赛道结果 **少于 10 篇** 时，可能提示：「xx 赛道 10w+ 文章较少，您可以拓展过去 30 天或者看看综合 10w+ 文章~」（以脚本输出为准）。  

---

## 注意事项与限制

- 本 Skill **仅输出榜单数据**，不提供代写、规律拆解或投放建议。  
- **禁止编造**作者、标题、阅读数、链接；必须以脚本返回为准。  
- HTML 内容须与对话表格**一致**；`display_count` 与展示条数对齐。  
- 遵守平台规范；外链与账号信息以脚本返回为准。
