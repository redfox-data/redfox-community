## 输出规范（最高优先级）

脚本运行后会生成 `./xhs_breaking_rankings.md`，该文件已包含全部格式化内容。

**必须执行**：读取 `./xhs_breaking_rankings.md` 全部内容，原样输出给用户。禁止自行编写、修改格式或省略任何部分。

**输出必须严格包含以下4个部分，按此顺序，缺一不可**：

1. **爆款笔记表格** — Markdown表格，含序号/笔记信息/互动总数/点赞数/评论数/收藏数/分享数/发布时间
2. **爆款规律分析** — 标题特征（Markdown表格）、内容主题（Markdown表格）、可参考亮点（列表）
3. **订阅服务** — 含订阅选项和订阅逻辑说明
4. **HTML文件包** — 将 `./xhs_breaking_rankings.html` 以文件包形式提供

**禁止**：

- 将表格改为列表或其他格式
- 省略订阅服务或HTML文件包
- 自行编写内容而非读取md文件
- 折叠任何部分
- 调整输出顺序
- 数据为空时自动查询其他日期

## 日期规则

**未指定日期**：用户未提及任何日期时，脚本按19:30规则自动计算：

| 当前时间 | 传入API日期 | 示例（今天4月24日） |
| -------- | ----------- | ------------------- |
| >= 19:30 | 前一天      | 4月23日             |
| < 19:30  | 前两天      | 4月22日             |

**模糊日期词**：用户说"今天"、"前天"等时，传 `--rank_date "今天"` 脚本自动转换为具体日期。

**明确日期**：用户指定具体日期时，传 `--rank_date "YYYY-MM-DD"`。

## 执行步骤

1. 用户触发时，先询问查询日期或分类，禁止自动生成
2. 用户输入后，调用脚本：
   - 未指定日期：`python scripts/fetch_explosive_articles.py --keyword "关键词" --top_n 50`
   - 指定日期：`python scripts/fetch_explosive_articles.py --rank_date "YYYY-MM-DD" --keyword "关键词" --top_n 50`
3. **数据为空时**：将脚本提示原样展示，等待用户确认，禁止自动查询其他日期
4. 脚本生成 `./xhs_breaking_rankings.md` 和 `./xhs_breaking_rankings.html`
5. **读取 `./xhs_breaking_rankings.md` 全部内容原样输出**
6. **将 `./xhs_breaking_rankings.html` 以文件包形式提供**
