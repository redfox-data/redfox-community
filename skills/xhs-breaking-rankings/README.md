# 小红书冷门爆款探测器 / xhs-breaking-rankings

---

## 简介

本 Skill 面向 **小红书低粉高互动笔记榜单**：按分类与日期筛选「粉丝低于 5000、互动大于 500」的冷门爆款样本，输出可对标学习的榜单、规律拆解与可视化页面，帮助你在头部账号之外提前发现潜力内容。

**核心价值**

- **低粉爆文筛选**：避开大 V 流量红利，小账号案例更具参考性。  
- **全分类覆盖**：支持 **25** 个主流内容分类，按口语关键词自动匹配赛道。  
- **规律自动拆解**：基于榜单提炼标题特征、内容主题与可复用亮点。  
- **双格式交付**：Markdown 表格 + HTML 可视化文件包，默认先展示 **TOP20**，可查看更多至 **TOP50**。  
- **可选订阅**：支持每日固定时间推送最新榜单（以产品侧开通为准）。  

**适用对象**

新人博主找对标、品牌运营追热点、MCN 挖掘潜力达人、内容团队做赛道调研。

**运行环境**：**Python 3**；须安装 **`requests>=2.28.0`**（见技能包 `dependency`）。无需小红书账号与 API 密钥。在已启用本 Skill 的 Agent 中通过自然语言触发。

---

## 功能特性

### 核心功能

- **分类与日期查询**：支持指定日期、口语日期（如「今天」）或未指定日期时按 **19:30** 规则自动匹配近端可查日。  
- **完整结果呈现**：含爆款笔记表、规律分析、订阅选项与 HTML 文件包，顺序与版式以当次生成为准。  
- **结构化榜单**：序号、笔记信息、互动总数及点赞/评论/收藏/分享、发布时间等字段。  
- **查看更多**：首轮常见展示 20 条，可说「查看更多」获取完整榜单（最多 **50** 条，以当次查询为准）。  

### 特色亮点

- **灵活日期适配**：明确日期传 `YYYY-MM-DD`；未指定时由脚本按当前时刻与 **19:30** 边界计算可查日。  
- **无额外账号门槛**：安装 Python 依赖即可使用主流程。  
- **数据为空不擅自改查**：某日期无数据时按提示处理，不自动换日期重查（除非你再指定）。

## 使用指南

### 你可以怎么说

| 你的说法 | 大致会得到什么 |
| --- | --- |
| 「查 2026-04-20 美妆类低粉爆款」 | 该日该分类榜单 + 规律分析 + 订阅问句 + HTML |
| 「今天 / 前天 出行类冷门爆款」 | 口语日期换算后查询 |
| 「查看更多」 | 展示当次查询剩余榜单条目（至多 50 条） |
| 「订阅」/ 回复数字 **1** | 开通每日推送（常见为 **19:30** TOP50，以产品为准） |
| 「取消订阅」 | 关闭每日推送 |

### 使用提示

- 首次触发时，Agent 可能会先确认你要查的**日期**或**分类**再执行查询。  
- 查询标准常见表述为：**粉丝数低于 5000、笔记点赞数大于 500**（以当次输出说明为准）。  
- 对话中看到的是格式化结果，而非脚本路径或源码。

## 重要数据说明

**筛选口径**

- 榜单条目通常满足：**粉丝 &lt; 5000**、**点赞 &gt; 500**（与接口及脚本说明一致；展示文案以当次输出为准）。  

**更新与日期**

- 数据常见为**每日 19:30** 更新昨日榜单口径。  
- **未指定日期**时：当前时间 **≥ 19:30** 多查**前一天**；**&lt; 19:30** 多查**前两天**（由脚本计算）。  
- 展示为查询时刻快照，与线上实时互动可能略有差异。  

---

## 注意事项与限制

- 本 Skill 提供**榜单与规律参考**，不保证后续流量表现。  
- 无数据时请勿要求自动改查其它日期，应按提示确认后再查。  
- 订阅推送是否可用、推送时间以产品侧为准。  
- 遵守小红书平台规范；外链与账号信息以榜单返回为准。
