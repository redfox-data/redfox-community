# TikTok账号搜索

## 📝 简介

输入关键词就能搜到 TikTok 上的博主账号，按粉丝数从高到低给你排好，昵称、粉丝量、个人简介一目了然，点一下链接就能跳转主页看详情。搜不到内容时会自动推荐 10 个相近关键词，帮你换着方向继续找。

## ✨ 核心价值

**内容新鲜不滞后：** 每次搜索都直接拉取平台当前的最新账号数据，不会拿旧缓存糊弄你。

**按粉丝数高低排好：** 搜索结果自动按粉丝量从高到低排列，头部大号一眼就能看到。

**搜不到自动扩词：** 关键词搜不出结果时，工具会一口气给你 10 个相近的推荐词，帮你换个角度继续挖。

**支持翻页浏览：** 结果多的时候可以一页页翻着看，每页 10 条，不遗漏任何一个账号。

## 🎯 适用对象

- 🔍 **内容创作者** — 找同赛道海外博主，研究对标账号的内容方向和运营思路。
- 📊 **运营 / 数据分析** — 快速摸清某个品类在 TikTok 上有哪些头部账号，掌握竞争格局。
- 🏢 **品牌 / MCN** — 筛选潜在合作博主，按粉丝量评估账号价值，辅助签约决策。
- 🌏 **跨境电商** — 搜索海外细分市场的达人资源，找到匹配的带货 / 推广人选。

## 🔄 工作流程

### Step 1：理解用户意图，提取关键词

**⚠️ 核心规则：语义理解优先，优先提取细分方向词，而非泛化大类词**

**1. 提取关键词**

- 从用户描述中提取 2~6 字的细分关键词
- 若用户意图模糊（如"帮我搜TikTok"），主动询问：「请问你想搜索哪个方向或领域的 TikTok 账号？」
- 不得在用户未提供关键词时擅自猜测并调用脚本

**2. 识别页码参数**（用户未指定时使用默认值）

- **页码**（`--page`）：默认 `1`
  - 用户提到"下一页"、"第2页" → 对应页码
  - 用户提到"上一页" → 当前页 - 1
  - 未提及页码 → `1`（默认）
- **翻页游标**（`--rid`）：第一页不传，翻页时必须传入上一页响应中返回的 `rid` 值

### Step 2：调用搜索脚本

```bash
python3 ~/.agents/skills/tiktok-account-search/scripts/search_tiktok_realtime.py "<关键词>" [--page 页码] [--rid 翻页游标]
```

**参数说明：**

| 参数     | 可选值     | 含义                         |
| -------- | ---------- | ---------------------------- |
| keyword  | 任意字符串 | 搜索关键词                    |
| `--page` | 正整数     | 页码，从1开始（默认1）        |
| `--rid`  | 字符串     | 翻页游标，第一页不传，后续页必传（从上一页响应 `rid` 字段获取） |

脚本以 JSON 格式输出：

| 字段       | 说明                           |
| ---------- | ------------------------------ |
| `accounts` | 匹配的账号列表（按粉丝数降序）  |
| `page`     | 当前页码                       |
| `has_next` | 是否有下一页（true/false）      |
| `total`    | 当前页返回条数                  |
| `rid`      | 翻页游标，下一页请求时传入 `--rid` |

每条账号字段：

| 字段             | 说明          |
| ---------------- | ------------- |
| `nickname`       | 账号昵称      |
| `account_id`     | TikTok 账号ID |
| `follower_count` | 粉丝数        |
| `signature`      | 个人简介      |
| `verified`       | 是否认证      |
| `region`         | 所属地区      |
| `profile_url`    | 主页链接      |

### Step 3：判断结果并展示

#### 情况 A：accounts 数量 > 0（有匹配结果）

**A1. 告知用户查询范围**

> 📊 关键词「**XXX**」查询到 **N 个** TikTok 账号（第 {page} 页），以下是详细数据：

**A2. 渲染 Markdown 表格（全部展示）**

```markdown
| #   | 账号昵称         | TikTok ID     | 粉丝数  | 简介摘要                     | 主页链接                    |
| --- | ---------------- | ------------- | ------- | ---------------------------- | --------------------------- |
| 1   | 昵称             | @account_id   | 305.2w  | Fitness Content Creator      | [访问主页](profile_url)     |
```

**数字格式化规则：**
- `< 10000`：原始数字（如 `320`）
- `≥ 10000`：`x.xw` 格式（如 `1.2w`）

**昵称规则：** 纯文本，超过 20 字截断并加 `...`

**简介规则：** 取 signature 字段，超过 40 字截断并加 `...`；为空时显示 `-`

**主页链接规则：** 固定显示文字 `[访问主页](profile_url)`，点击可跳转至 TikTok 账号主页

**A3. 翻页提示（⚠️ 紧接在表格之后，不可省略，每次必须输出）**

- 若 `has_next` 为 true：
> 📄 当前第 **{page}** 页。回复「下一页」继续查看。

- 若 `has_next` 为 false（已是最后一页）：
> 📄 当前第 **{page}** 页，已无更多数据。

**翻页时「下一页」调用规则：** 若用户回复「下一页」，调用脚本时必须在上一页响应的 `rid` 基础上递增页码：
```bash
python3 ~/.agents/skills/tiktok-account-search/scripts/search_tiktok_realtime.py "<关键词>" --page {page+1} --rid "{上一页rid}"
```

**⚠️ A1~A3 缺一不可，必须在同一轮输出中连续完成。若缺少 A3，视为执行错误，必须重新补充输出。**

#### 情况 B：accounts 数量 = 0（无匹配结果）

**B1. 抱歉提示 + 拓词推荐**

> 😔 抱歉，未找到与「**XXX**」直接相关的 TikTok 账号，你可以尝试用更短或更宽泛的关键词重试（扩展词1,扩展词2,...扩展词10）

AI 必须生成**固定 10 个**扩展词，2~6 字，英文逗号分隔，不得少于 10 个。

**⚠️ B1 必须在同一轮输出中完成，不得中断。**

### Step 4：结束输出

全部内容展示完毕后，不等待、立刻结束输出。
