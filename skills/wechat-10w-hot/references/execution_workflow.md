# 公众号 10w+ — 执行流程分步说明

本文档为 `wechat-10w-hot` Skill 的**分步执行细则**（意图 → 脚本 → 规律分析 → 订阅 → HTML → 订阅回复），与根目录 `SKILL.md` 中的「核心执行规则」配合使用；执行完整流程前请通读本文件。

---

## 步骤 1：意图识别

- **总榜触发词示例**：今日爆文、10w+、爆文推荐、最新爆文、热门爆文、每日爆文、爆文榜单等 → 走总榜参数。
- **领域**：出现明确领域词（如 AI 编程、财经、健康饮食）→ 提取为 `keyword`。
- **模糊**：引导补全领域，或确认后走总榜。

---

## 步骤 2：获取数据（脚本）

### 总榜参数集

| 项               | 值                                                        |
| ---------------- | --------------------------------------------------------- |
| `keyword`        | `""`                                                      |
| `source`         | `公众号10w+阅读文章推荐`（与 `fetch_hot_articles.py` 默认一致；若有专项约定见 `api-spec.md`） |
| `start_date`     | `daybeforeyesterday`（**严禁**用 `yesterday` 顶替本场景） |
| `top_n`          | `50`                                                      |
| `mode` / `limit` | 首轮 `preview` + `limit 10`                               |

### 领域参数集

| 项               | 值                                 |
| ---------------- | ---------------------------------- |
| `keyword`        | 用户领域词                         |
| `source`         | 同上固定值                         |
| `start_date`     | 用户指定 → `YYYY-MM-DD`；无则 `""` |
| `top_n`          | `50`                               |
| `mode` / `limit` | 首轮 `preview` + `limit 10`        |

### 分页

- 首轮：`--mode preview --limit 10`
- 展开：`--mode full`
- 若总数超过 10：提示「还有 X 条数据未展示，可继续查看」

### 候选与展示

1. `temp_articles.json` 由脚本写入，供 `generate_hot_html.py` 使用。
2. **对话正文 = 脚本 stdout**：preview / full 印出什么就展示什么，**不得删行、不得改表格、不得只贴概览不贴详情**。
3. **禁止凑数、禁止编造**；无数据时按根目录 `SKILL.md` 常见问答处理。

### 内容分析（四维度，写作要求）

每条详情中的「内容分析」须覆盖且融合为一小段：**内容概述、热点利用、传播作用、达成效果**。

### 输出形态速查

概览表（列：序号、标题、作者、阅读数）；详情块：

```markdown
**序号、[标题](文章链接)**
📄 作者：[名称](https://open.weixin.qq.com/qr/code?username={accountId})
👀 阅读数：…
⏰ 发布时间：…
🔍 内容分析：…（四维度融合）

---
```

## 步骤 3：爆款规律分析

在步骤 2 **全部贴完后**输出，结构建议：

```markdown
📊 爆款规律分析

标题特征：

- …

内容主题：

- …

发布时间：

- …

公众号特征：

- …
```

要求：**不计算占比**、用**当前榜单真实标题/时间**举证；禁止复述文档内置长示例当结论。

---

## 步骤 4：订阅询问

固定话术块：

```markdown
📬 订阅服务

订阅推送 - 每天19点30分推送最新10w+公众号爆文
暂不需要 - 仅本次查询
```

仅询问，**不**在本步实现推送。

---

## 步骤 5：生成 HTML

- 脚本：`scripts/generate_hot_html.py`
- 常用参数：`--temp_file temp_articles.json`（默认即此）、`--output "{关键词}_热门榜单.html"`、`--display_count <与对话一致>`
- 首轮示例：`--display_count 10`；全文：`50`；若总数少于 10 条则用实际 N。

**生成后必须**：`preview_url` 预览 → 告知路径 → 问是否调样式。

### HTML / PDF 视觉规范（摘要）

| 类别   | 要求                                                                                                                                                                                                             |
| ------ | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 色     | 主色 `rgb(0,179,84)`；链接与阅读数 `#0088ff`；背景 `#fff`                                                                                                                                                        |
| 布局   | 序号 🥇🥈🥉 或数字在标题左；用户 \| 类型 \| 阅读 \| 时间 竖线分隔且竖线视觉缩短居中                                                                                                                              |
| 头图   | **不要**「爆款研究院」等品牌头                                                                                                                                                                                   |
| 图标   | 用户 `👤`、阅读 `📖`、时间 `📅`                                                                                                                                                                                  |
| 分析区 | 无左边框；白底 + 浅绿辅助底                                                                                                                                                                                      |
| PDF    | html2pdf.js；按钮在标题右、蓝渐变白字 12px；`pagebreak: { mode: 'none' }`、`margin: 0`、按内容算纸张；`page-break-inside: avoid` / `break-inside: avoid` 作用于 `article-item`、`article-list`、`content` 等容器 |
| 页脚   | 「公众号10w+阅读文章，每日更新最新爆文内容」+「备注：互动数据为入库快照，实时数据可能持续增长」                                                                                                                  |

---

## 步骤 6：订阅回复处理

- **订阅推送**：引导输入领域词 → 再跑领域逻辑（TOP50 + 规律）；说明定时推送侧用 **`--start_date yesterday`**（与总榜 `daybeforeyesterday` **不同**）。
- **暂不需要**：结束本轮。
- 用户可随时「取消订阅」类指令按产品约定处理（若有）。
