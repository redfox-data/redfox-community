# 公众号 10w+ 热门文章推荐 / wechat-10w-hot

---

## 简介

本 Skill 面向 **公众号全站热点扫榜、垂类 10w+ 爆文选题、运营侧固定节奏参考**：从数据源拉取 **10w+ 阅读**量级的热门文章榜单，在对话中给出结构化概览与分条详情，并支持生成公众号风格的 **HTML** 页面，便于浏览器内导出 **PDF**（单页、无强制分页等规范以执行说明为准）。

**核心价值**

- **总榜 / 领域**：支持全站热门（空关键词）与按领域词查询；后台常见拉取规模为 **TOP50**，首轮对话可仅预览前 **10** 条，确认后再展开全文。  
- **结构化交付**：数据说明 → 概览表 → 分条详情（含「内容分析」等维度）→ 统计，**顺序与块结构以当次脚本输出为准**，便于直接用作选题与对标材料。  
- **HTML / PDF**：在完整流程下可生成与当前对话展示条数一致的 HTML 文件，便于分享与归档。  
- **订阅引导**：在规律分析后可按固定话术询问是否订阅垂类推送（具体承接逻辑以包内执行文档为准）。  

**适用对象**

公众号编辑与运营、内容策划与增长、需要「可核对样本」的 10w+ 爆文参考的创作者或团队。

**运行环境**：需 **Python 3**；拉榜脚本使用 **`coze_workload_identity`** 提供的请求能力（见 `scripts/fetch_hot_articles.py`）；HTML 生成脚本以标准库为主（见 `scripts/generate_hot_html.py`）。在已启用本 Skill 的 Agent 客户端中通过自然对话触发。

## 功能特性

### 核心功能

- **全站热门（总榜）**：识别「今日爆文」「10w+」「最新爆文」等说法时，按包内约定使用 **`daybeforeyesterday`** 等日期语义拉取总榜（**勿**用 `yesterday` 顶替「今日/最新」总榜场景）。  
- **领域查询**：从用户话术中抽取领域词，与总榜共用 **TOP50 + 首轮预览** 模式；`start_date` 可为具体日期、`""`（约最近 30 天）或约定特殊值。  
- **分页预览**：首轮常见为 **preview + 前 10 条**；总行数超过 10 时可提示继续查看，用户确认后以 **full** 模式展示全部。  
- **配套产物**：运行过程中可产生 **`temp_articles.json`** 与 **`{关键词}_热门榜单.html`**（文件名可带语义自定义，以实际执行为准）。  

### 特色亮点

- **输出与脚本一致**：对话中的榜单与详情应以 **`fetch_hot_articles.py` 的标准输出**为准完整呈现，避免只贴表、不贴详情或擅自改写字段。  
- **规律分析后置**：爆款规律分析应在**榜单全文展示完成后**再撰写，且须基于**当前榜单事实**，避免套用与本次数据无关的示例结论。  
- **HTML 条数对齐**：生成 HTML 时的 **`--display_count`** 须与**本轮对话实际已展示的条数**一致（如 10 / 50 / 实际 N）。  
- **时间语义区分**：用户口头「今日/最新」总榜与**订阅定时推送**场景下，对 `daybeforeyesterday` / `yesterday` 等取值有不同约定（见「重要数据说明」）。

## 使用指南

### 你可以怎么说

| 你的说法 | 大致会得到什么 |
| --- | --- |
| 「今日爆文」「10w+」「热门爆文榜单」 | 总榜路径：按约定日期参数拉取，首轮常见为 **前 10 条预览**，随后可接规律分析与订阅问句，并生成 HTML |
| 「推荐 xx 领域 10w+」 | 抽取领域词查询，首轮同样多为预览 **10 条**，可按提示展开查看余下条目 |
| 「把后面几十条也展示」 / 「全部展示」 | 在预览后触发 **full** 模式，对话与 HTML 的展示条数与 **`--display_count`** 对齐 |
| 说法较模糊，如「看看热门」 | 助手可能追问领域词，或在征得同意后走总榜 |

### 使用提示

- 需要**严格按步骤执行**（含订阅话术与 HTML 生成时机）时，请先让执行方阅读 **`references/execution_workflow.md`**，并与根目录 **`SKILL.md`** 中的「核心执行规则」一并遵守。  
- **「今日/最新」总榜**与**订阅推送**对日期特殊值（`daybeforeyesterday` / `yesterday`）的约定不同，不要混用。  
- 若某领域或日期下**无数据**，应如实提示并建议更换更热门领域词等，**不要虚构文章条目**。

## 重要数据说明

**同步与快照**

- 列表更新节奏常见为 **每日 19:30** 一类口径（以脚本首段「数据说明」文案为准）；展示为**拉取时刻快照**，与线上实时阅读数可能不一致。  
- 数据覆盖常见为**最近约 30 天**；超出范围时由接口或脚本按规则截断或报错。  

**`start_date` 语义（须与脚本及执行文档一致）**

| 取值 | 含义（用户侧理解） |
| --- | --- |
| `daybeforeyesterday` | 用于用户主动要「今日 / 最新」等**总榜**场景时，换算为前天对应的 `YYYY-MM-DD`，以与库同步节奏闭合 |
| `yesterday` | **主要用于订阅定时推送**等场景（与总榜「今日最新」语义区分） |
| `YYYY-MM-DD` | 用户指定的单日 |
| `""` | 不限定单日，领域查询时常见为最近约 30 天 |

## 注意事项与限制

- 本 Skill 提供**数据与版式参考**，不构成刷量、买阅读或违反平台规则的操作建议。  
- **禁止**在未实际执行脚本的情况下编造榜单；**禁止**只展示表格而省略脚本输出的详情块。  
- **规律分析**须排在全文展示之后，且不得用文档或历史对话中的示例文章代替本次真实数据作结论。  
- **HTML 生成**在常见流程中应在订阅问句发出后按文档执行；**`--display_count`** 必须与当前对话已展示条数一致。  
- 产出路径与文件名（如 `temp_articles.json`、`*_热门榜单.html`）以运行环境为准，如需留存请自行备份。

## 常见问答（FAQ）

**Q：用户说「今日爆文」时，`start_date` 应用哪个特殊值？**  
A：总榜场景用 **`daybeforeyesterday`**；不要用 **`yesterday`** 来充当「今日/最新」总榜。

**Q：订阅推送场景里日期怎么取？**  
A：定时推送侧按执行文档使用 **`yesterday`**，与上一条总榜语义区分。

**Q：可以只贴概览表、不贴每条详情吗？**  
A：**不可以**。须将脚本标准输出中的各块按原顺序完整呈现。

**Q：JSON 里有 50 条，但对话只展示了 10 条，HTML 应生成几条？**  
A：**10 条**；`--display_count` 以**对话实际已展示条数**为准。

**Q：没有数据时怎么说？**  
A：提示未查询到相关 10w+ 数据，并建议更换更热门领域词等；**不虚构**文章。
