# 公众号 10w+ 热门文章推荐 / wechat-10w-hot

---

## 简介

本 Skill 面向 **公众号 10w+ 阅读爆文**场景：按 **标准分类** 与时间窗口拉取榜单，在对话中给出完整 **Markdown 榜单**（数据说明 → 概览表 → 分条详情 → 统计），并生成可导出 **PDF** 的 **HTML** 页面；支持在规律分析后引导按赛道 **订阅推送**（是否已开通以产品侧为准）。

**核心价值**

- **总榜 / 分赛道**：覆盖 **23** 个标准分类及 **总排名**；常见候选规模 **TOP50**，首轮对话可 **preview** 前 **10** 条，确认后可 **full** 展示全部。  
- **数据诚实**：榜单正文须来自 **`fetch_hot_articles.py` 的 stdout**，**禁止**编造文章或阅读数。  
- **HTML + PDF**：榜单页遵循微信绿视觉规范，**`--display_count`** 与对话已展示条数一致，支持浏览器侧 **html2pdf.js** 导出。  
- **规律复盘**：在榜单展示完成后，基于**当前真实条目**输出爆款规律分析，并询问是否订阅赛道。  

**适用对象**

公众号编辑与运营、内容负责人、增长与商务等需要「可核对、可分享、可打印」10w+ 样本的角色。

**运行环境**：**Python 3**；拉榜脚本以仓库内 **`scripts/fetch_hot_articles.py`** 为准（标准库网络请求）；HTML 生成见 **`scripts/generate_hot_html.py`**。在已启用本 Skill 的 Agent 中通过自然对话触发。

## 功能特性

### 核心功能

- **榜单拉取**：`scripts/fetch_hot_articles.py` → 写入 **`temp_articles.json`** → 对话内 **原样**输出 stdout。  
- **分类与时间**：用户口语映射到标准 **`--type`**（见 **`references/category-mapping.md`**）；**`--start_date` / `--end_date`** 须结合 **每日 18:30** 同步节奏与当前钟点计算（见 **`references/time-and-date-rules.md`**）。  
- **预览与全量**：默认 **`--mode preview`**、**`--limit 10`**；用户要求「展开全部 / 看 50 条」时改为 **`--mode full`**，HTML 条数同步。  
- **固定节奏**：识别意图 → 跑脚本 → 规律分析 → 订阅询问 → **立即**生成 HTML（**不等**订阅回复）→ 用户若订阅再走下一分支（细则见 **`references/agent-workflow.md`**）。  

### 特色亮点

- **脚本输出即正文**：概览表、详情块、统计区**不可删改重排**。  
- **18:30 分水岭**：早于 **18:30** 与晚于 **18:30**，「最新可用日期」不同；用户说的「今天 / 昨天」与真实查询区间不一致时，须使用 **`references/time-and-date-rules.md`** 中的**固定致歉话术**。  
- **禁止凑数**：少于 10 条如实展示；无数据时按参考文档提示改查总榜或其它分类，**不虚构**条目。

## 重要数据说明

**同步与快照**

- 榜单库 **每日 18:30** 对齐前一日数据；对外推送话术常见 **19:30** 口径（以脚本首段「数据说明」为准）。  
- 展示为**拉取时刻快照**，与线上实时阅读数可能不一致。  
- 用户说「最近 / 最新」且无明确日期时，默认 **过去 7 天** 区间。  

**时间不一致（须原文使用）**

凡用户问的时间与**实际查询落库日期**不一致，在「数据说明」后追加 **`references/time-and-date-rules.md`** 中对应固定话术（如「今天的数据要明天才会更新…」「昨天的数据还未更新…」「最多支持回溯过去30天…」等）。

**脚本日期关键字（辅助）**

`fetch_hot_articles.py` 支持将 **`yesterday`**、**`daybeforeyesterday`**、`today`、`YYYY-MM-DD` 等解析为具体日期；**Agent 仍须**按 18:30 规则与区间表计算 **`--start_date` / `--end_date`**，不可仅凭口语随意填参。

## 注意事项与限制

- 本 Skill 提供**数据与版式参考**，不构成刷量、买阅读或违反平台规则的操作建议。  
- **禁止**未跑脚本就给出例文榜单；**禁止**只贴概览表、省略详情与统计块。  
- **规律分析**须在榜单全文展示之后，且仅基于**本次脚本数据**。  
- **HTML**：`generate_hot_html.py` 的 **`--display_count`** 必须等于对话已展示条数。  
- 产出如 **`temp_articles.json`**、HTML 文件名以运行环境为准，请自行备份。

## 常见问答（FAQ）

**Q：能否不跑脚本直接给几篇例文？**  
A：**不能**。无脚本 stdout 即视为未按本 Skill 执行。

**Q：用户只要 HTML、不要长文？**  
A：仍须**先完整展示**脚本 stdout，再生成 HTML。

**Q：`temp_articles.json` 有 50 条但对话只展示了 10 条，HTML 几条？**  
A：**10 条**；`--display_count` 以**对话实际展示**为准。

**Q：接口 URL 以哪里为准？**  
A：以 **`references/api-spec.md`** 与 `fetch_hot_articles.py` 内常量为准；若有差异以脚本为准并应同步修正文档。

**Q：没有数据时怎么说？**  
A：按脚本与参考文档如实提示，建议调整分类或时间范围；**不虚构**文章。
