# 小红书爆款标题生成器 / xhs-explosive-content-suite

---

## 简介

本 Skill 面向需要在**小红书**侧提高标题点击率与种草效率的创作者与运营：先按关键词**获取**热门笔记样本，完成「爆款标题分析」后，再一次性产出 **10 条**可落地的标题方案（含匹配指数、参考链接与推荐理由等固定版式）。

**核心价值**

- **数据对齐**：基于脚本拉取的样本做分析，避免凭空编造互动与案例。  
- **先析后写**：必须先输出「爆款标题分析」，再生成新标题，规律可追溯。  
- **批量交付**：统一版式输出 10 条，便于对比选用与 A/B。  

**适用对象**

新人博主起号、品牌与电商运营写笔记标题、MCN 批量化提案等。**完整执行细则**以同目录 **`references/core_workflow.md`** 为准；本文档为面向用户的摘要。

**运行环境**：**Python 3**；依赖 **`requests>=2.28.0`**（见 `SKILL.md` frontmatter）。在已启用本 Skill 的 Agent 中通过自然对话与脚本调用配合使用。

---

## 功能特性

### 核心功能

- **关键词驱动**：支持产品词、话题或泛化大类；泛化大类须先走拓展确认流程（见使用指南）。  
- **时间策略**：数据不足时仅自动扩展时间窗（近 1→3→7→30 天），**不换词**。  
- **双阶段产出**：爆款标题结构分析 → 再创作 10 条新标题（非照抄原题）。  
- **固定交付形态**：每条含匹配指数（8–10、一位小数）、参考爆款链接、推荐理由与分隔线，与 `core_workflow` 校验清单一致。  

### 特色亮点

- **形态边界清晰**：用户输入的产品形态与查询关键词对齐，不擅自扩展到相邻品类。  
- **用语合规**：对用户可见表述避免使用「爬取」「抓取」，统一为「获取」；敏感原标题须中性概括。

## 使用指南

### 你可以怎么说

| 你的说法 | 大致会得到什么 |
| --- | --- |
| 「帮我起几个防晒霜的小红书标题」 | 抽取关键词 → 按日期规则跑脚本 → 读 `关键词_爆款数据.md` → 先分析再出 10 条 |
| 「美妆方向标题」（大类） | 先给细分方向，**等待**你回复「拓展」或「不拓展」后再查询 |
| 「全站热门里找灵感起标题」 | 在 workflow 允许下使用空关键词等总榜路径，再进入分析与生成 |

### 命令速查（在技能根目录执行）

| 命令示例 | 功能 |
| --- | --- |
| `python scripts/fetch_xhs_trends.py --keyword "防晒霜" --start-date 2026-04-01` | 按关键词与起始日获取数据并写 Markdown 报告 |
| `python scripts/fetch_xhs_trends.py --keyword "a,b,c" --start-date 2026-04-01` | 多关键词（英文逗号分隔） |
| `python scripts/fetch_xhs_trends.py --keyword "" --start-date 2026-04-01` | 无赛道关键词时的全站热门路径（以 workflow 为准） |

### 使用提示

- **顺序不可颠倒**：必须先「爆款标题分析」，再「生成新爆款标题」。  
- **数据不足**：只扩时间窗、不换词；扩至近 30 天仍不足则如实说明，**禁止编造**标题、互动数与 `photoId` 链接。  
- **泛化词**：助手给出细分推荐后，须**停止等待**你的「拓展 / 不拓展」回复，**禁止**同轮未等回复就执行查询。

## 使用场景

| 场景 | 角色 | 需求描述 | 使用方式 |
| --- | --- | --- | --- |
| 新品起标题 | 电商 / 品牌运营 | 有卖点但不会写小红书风格标题 | 给产品词与卖点 → 拉数据 → 先分析再出 10 条备选 |
| 账号日常更新 | 博主 / 编导 | 同赛道缺选题与标题套路 | 输入赛道或细分词 → 按数据规律迭代标题库 |
| 大类试水 | 新手创作者 | 只想到「美妆」「穿搭」等大类 | 走泛化词推荐 → 选拓展或不拓展后再查询 |
| 全站追热点 | 运营 | 想看近期全站热门再改编 | 在 workflow 约定下查全站 → 再落到具体创作 |

## 注意事项与限制

- **数据**：来自脚本约定接口与入库快照，**非实时**；互动与链接以报告为准，**禁止虚构**。  
- **任务归属**：全流程宜由**主 Agent** 执行，以免格式与顺序被破坏（见 `SKILL.md`）。  
- **合规**：遵守社区规范与广告合规习惯；参考链接应可溯源。  
- **范围**：不替代平台审核；发布前仍须自行复核合规。
