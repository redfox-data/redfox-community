# 抖音每日热门作品榜

## 1. 简介

**一句话定位**：抖音每日最热作品榜查询工具，日度收录全平台抖音作品，输出单日点赞TOP50榜单，支持28个赛道分类查询和历史回溯。

**核心价值**：
- **日度精准更新**：每日人工收录核验全平台万级抖音作品，每日06:00更新昨日数据
- **多维赛道覆盖**：支持28个赛道分类查询，从全品类到细分领域全覆盖
- **灵活时间回溯**：最多回溯过去30天，支持历史数据对比分析

**适用对象**：抖音内容创作者、MCN机构运营、品牌营销策划、抖音数据分析师。

## 2. 功能特性

### 核心功能

| 功能模块 | 能力描述 | 核心价值 |
|----------|----------|----------|
| 📊 全品类日榜 | 单日点赞TOP50榜单 | 快速了解抖音最热内容 |
| 🏷️ 赛道分类查询 | 28个细分赛道独立查询 | 精准定位垂直领域热点 |
| 📅 历史回溯 | 最多回溯30天 | 对比分析历史趋势 |
| 👤 达人追踪 | 展示作者粉丝数 | 发现潜力达人账号 |
| 📈 互动数据看板 | 收藏/评论/分享/点赞 | 多维度衡量作品表现 |
| ⏰ 订阅推送 | 每日06:00自动推送 | 持续追踪，不错过热榜 |

### 特色亮点

- **超链接标题**：所有作品标题以 `[标题](workUrl)` 格式输出，可直接点击跳转抖音
- **智能时间判断**：自动判断数据可用性，当日/未来日期自动回退到最近可用数据
- **TOP20默认展示**：默认展示TOP20，可按需查看完整50条
- **赛道未匹配提示**：自动列出28个赛道供用户选择

## 3. 一键安装

### 鉴权

#### 获取 API Key

请前往 [红狐hub](https://redfox.hk/settings/api-keys?source=github) 获取API KEY

#### 配置 API Key

方案1: 以OpenClaw为例，将REDFOX_API_KEY添加到~/.openclaw/openclaw.json中，部分内容如下：

```bash
{ "env": { "REDFOX_API_KEY": "ak_xxxx..." } }
```

方案2: 终端配置：

```bash
export REDFOX_API_KEY="ak_xxxx..."
```

### 依赖安装

本技能使用API直调方式，无需额外Python依赖。如需调用脚本，请确保Python 3可用。

### 环境变量配置

| 变量名 | 说明 | 必填 |
|--------|------|------|
| `REDFOX_API_KEY` | 红狐 API Key | 是 |

## 4. 使用指南

### 基础使用

#### 查询昨日全品类日榜（默认行为）

无特殊说明时，默认返回昨日全品类日榜TOP20。

**接口调用**：
- 接口地址：`POST https://redfox.hk/story/api/dy/search/likesRank`
- 认证方式：请求头 `X-API-KEY`，值从环境变量 `REDFOX_API_KEY` 获取
- 可选参数：`type`（赛道）、`startTime`、`endTime`（日期范围）

#### 查询指定赛道

用户可指定以下28个赛道：

```
全部、个人才艺、生活vlog、财富理财、二次元、居家装修、学习教育、小剧场、
数码科技、旅行、美食、化妆美容、动物、亲子、汽车、情感、三农、健康医学、
潮流风尚、舞蹈才艺、颜值造型、人文、音乐、影视、身体锻炼、体育、明星娱乐、游戏
```

未能匹配时明确告知并列出全部赛道，询问用户选择。

### 高级使用

#### 时间判断规则

| 用户输入 | 处理方式 |
|----------|----------|
| 今天 / 当前 / 最新 | 回复说明最新为昨日，提供昨日数据 |
| 未来日期 | 回复提示，提供昨日数据 |
| 30天内历史日期 | 直接查询对应日期 |
| 超过30天前 | 回复提示，提供最接近时间范围内的数据 |

**时间提示语**：
- 当日/未来：「非常抱歉🙏，我们最新的是昨日数据，将为您提供最接近您需求的昨日热榜。」
- 超出回溯：「非常抱歉🙏，目前榜单最多支持回溯「过去30天」，我将为您查询最接近您需求的时间范围~」

#### 标准输出格式

榜单以 Markdown 表格呈现，**作品标题必须使用 `[标题](workUrl)` 格式输出为可点击的超链接**。各列说明：

| 列名 | 来源字段 | 说明 |
|------|---------|------|
| 排名 | rank | 当日点赞排名 |
| 作品标题 | video_title + workUrl | **必须展示为 `[标题](workUrl)` 超链接格式** |
| 作者（粉丝数） | author_name + author_fans | 账号名 + 粉丝总数 |
| 赛道 | category | 仅全品类查询时展示，查询特定赛道时隐藏此列 |
| 收藏 | collect | 入库时快照 |
| 评论 | comment | 入库时快照 |
| 分享 | share | 入库时快照 |
| 点赞 | like | **入库时快照，数值需加粗显示** |
| 发布时间 | publish_time | 格式 MM-DD HH:00 |

**输出模板**：

```markdown
💡 榜单说明：每日 06:00 更新昨日数据。互动数据为入库时间，与实时数据存在差异。

📊 抖音每日点赞TOP20（YYYY-MM-DD）

| 排名 | 作品标题 | 作者（粉丝数） | 赛道* | 收藏 | 评论 | 分享 | **点赞** | 发布时间 |
|------|---------|--------------|-------|------|------|------|------|---------|
| 1 | [红烧肉的教程](https://www.douyin.com/video/xxx) | 美食达人（10w+） | 美食 | 10w+ | 10w+ | 10w+ | **10w+** | 05-28 12:00 |

*查询特定赛道时去掉"赛道"列

⚡ 更多操作
• 本次榜单完整共50条数据，是否需要查看剩余30条？

📬 订阅服务
1️⃣ 我们每日 06:00 更新昨日数据，是否需要订阅每日的抖音账号最新排名，订阅后定时推送给您
2️⃣ 是否需要订阅具体赛道的账号表现？我们支持：[列出28个赛道]
```

#### 作品标题超链接规范（必须遵守）

- 作品标题**必须**以 Markdown 链接格式输出：`[作品标题](workUrl)`
- `workUrl` 为接口返回字段，是作品在抖音的访问链接
- **禁止**使用 HTML `<a>` 标签或终端 OSC 转义序列
- 即使终端不支持直接点击，Markdown 链接格式也能确保对话界面正确渲染为可点击超链接

#### 分页展示策略

- 默认展示 **TOP20**，用户明确要求查看完整榜单时再输出全部50条
- 数据不足20条时展示实际数量

#### 订阅服务

- 支持用户选择赛道和时间偏好
- 按每日 06:00 更新后自动推送
- 每个分类最多返回 50 条

详见 [api-config.md](api-config.md) 和 [interaction-guide.md](interaction-guide.md)。

## 5. 使用场景

### 场景一：内容创作者选题参考

**角色**：抖音短视频创作者
**需求**：了解昨日最火的作品类型和内容方向，为自己的创作提供参考
**使用方式**：查询全品类日榜TOP20，重点关注高赞作品的标题、赛道和内容特征
**预期收益**：快速把握抖音内容风向，提高作品爆款概率

### 场景二：MCN机构达人监测

**角色**：MCN运营
**需求**：监测旗下达人在各自赛道中的排名表现，发现同行竞品动态
**使用方式**：按赛道分别查询，对比不同达人的互动数据和排名变化
**预期收益**：精准评估达人表现，及时调整运营策略

### 场景三：品牌营销内容分析

**角色**：品牌营销策划
**需求**：分析特定赛道（如美食、旅行）的热门作品，寻找品牌植入方向
**使用方式**：查询目标赛道TOP20，关注达人粉丝规模、内容形式和互动数据
**预期收益**：找到与品牌调性匹配的达人，策划高效的种草内容

### 场景四：账号冷启动方向定位

**角色**：新入局抖音创作者
**需求**：了解不同赛道的竞争程度和爆款特征，选择适合的创作方向
**使用方式**：连续查询多个赛道近7天数据，对比各赛道门槛和上限
**预期收益**：基于数据做出赛道选择决策，降低试错成本

## 7. 常见问答

### 安装相关

**Q: 需要安装什么依赖？**
A: 本技能通过直接调用Redfox API获取数据，无需额外Python依赖。只需配置 `REDFOX_API_KEY` 环境变量即可。

**Q: 如何验证API Key配置成功？**
A: 执行一次全品类日榜查询，如果能正常返回数据则配置成功。

### 使用相关

**Q: 为什么查询"今天"的数据返回的是昨天的？**
A: 数据每日06:00更新昨日数据，当日的最新数据要到次日06:00才可用。系统会自动回退并提供最接近的数据。

**Q: 支持查询多久之前的数据？**
A: 最多回溯过去30天。

**Q: 每个赛道最多返回多少条？**
A: 每个分类最多返回50条。默认展示TOP20，用户要求时可展示完整50条。

**Q: 为什么有些赛道查不到数据？**
A: 可能是该赛道在查询日期没有收录到数据，建议尝试近邻日期或全品类查询。

### 故障排除

**Q: 查询返回空数据？**
A: 可能原因：1) API Key 无效或过期；2) 查询日期已超过30天回溯范围；3) 赛道名称拼写错误。请逐一排查。

**Q: 作品链接无法点击？**
A: 确保输出格式为 `[作品标题](workUrl)` Markdown 链接格式，不要使用 HTML 标签。
