# 小红书内容生成 / xhs-copywriter

---

## 简介

**xhs-copywriter** 面向小红书持续产出：把「找对标 → 拆规律 → 写正文」串成一条流水线——按关键词**获取**热门样本，提炼可复用结构，再输出可直接发布的标题、正文与标签等组合（具体条数与版式以执行时工作流为准）。

**核心价值**

- **多维样本**：围绕低粉高赞、当日点赞、当日增长、持续增长等维度组织参考（以 `references/core_workflow.md` 为准）。  
- **规律到文案**：从标题、开头、节奏、互动设计等角度提炼要点，再生成完整笔记文案。  
- **来源可追溯**：结果侧附带参考笔记链接与数据说明（规律可核对）。  

**适用对象**

个人博主、品牌运营、MCN 等需要稳定产出小红书笔记文案的角色。完整步骤与禁止项见 **`references/core_workflow.md`**。

**运行环境**：**Python 3**；拉数脚本以仓库内 **`scripts/fetch_xhs_trends.py`** 为准（实现以标准库网络栈为主，无需单独安装 `requests` 时以脚本实际 import 为准）。

---

## 功能特性

### 核心功能

- **多维爆款数据采集**：覆盖多类榜单维度，扩大样本覆盖面（细则见工作流）。  
- **爆款公式提炼**：从标题模式、开头结构、内容节奏、互动设计等维度拆解；对用户呈现提炼结论而非原始大表。  
- **完整文案输出**：推荐标题多条、正文、标签列表与公式来源说明等（条数与结构以工作流为准）。  
- **热点融入**：在流程允许范围内结合近期行业动态与热词（以实际执行为准）。  

### 特色亮点

- **风格自适应**：可提供个人写作片段作为语气参考。  
- **低配置门槛**：在常见部署下通过对话 + 脚本即可完成主路径（具体鉴权与网络以环境为准）。  
- **结论可核对**：输出附带参考链接与数据来源说明。

## 使用指南

### 你可以怎么说

| 你的说法 | 大致会得到什么 |
| --- | --- |
| 「帮我写一篇关于夏日防晒的小红书笔记」 | 进入话题确认 → 拉数拆规律 → 输出标题 + 正文 + 标签等 |
| 「帮我找 [话题] 的爆款笔记」 | 以该话题为关键词获取样本并展示提炼结论 |
| 「分析 [领域] 的爆款规律」 | 输出该领域的结构化规律总结 |
| 「生成 [话题] 方向的文案」 | 与「写笔记」同类，可接受略模糊的话题输入 |

### 使用提示

- 可提供几段自己常写的文字，便于语气贴近。  
- 工作流要求**不将原始爆款数据表直接甩给用户**，只呈现结论与可用文案。  
- 若接口失败或样本不足，应如实说明，**禁止编造**笔记标题与互动数据。

| 场景 | 角色 | 需求描述 | 使用方式 |
| --- | --- | --- | --- |
| 新人找方向 | 0 粉新手 | 不知道写什么、只看大号难模仿 | 输入领域关键词，获取可模仿的结构与文案 |

| 规律研究 | 内容策略 | 了解某领域易出圈结构 | 发起「分析 XX 领域爆款规律」类请求 |

## 注意事项与限制

- **全流程**：在标准流程下由执行方自动完成「获取数据 → 拆规律 → 补热点 → 生成」等步骤，用户侧以自然语言驱动即可。  
- **数据与合规**：内容须符合小红书社区规范；不含违规导流、虚假宣传等风险表述（生成后仍建议人工复核）。  
- **原始数据**：不向终端用户直接铺开原始抓取大表，仅呈现提炼结果与可用文案。
