# 小红书爆款标题打分器

## 1. 简介

基于小红书爆款数据分析的标题评分工具 -- 输入待评分的标题和相关主题关键词，系统自动查询同赛道爆款数据，提炼可落地的爆款规律，从 6 个维度（主题匹配度、结构合规度、利益清晰度、情绪唤醒度、稀缺性感知、合规安全性）对用户标题进行加权评分，输出 S/A/B/C 等级评分报告和优化建议。

**适用对象**：小红书创作者、内容运营、文案策划、需要验证标题质量的写作者。

---

## 2. 功能特性

**核心功能**：

- 多维度加权评分 -- 6 维度加权系统（主题 15%、结构 20%、利益 25%、情绪 20%、稀缺 15%、合规 5%），满分 10 分
- 爆款规律提炼 -- 从召回的实际爆款中自动提炼可落地的规律（结构 / 用词 / 利益点）
- 等级评定 -- S 级（9.0+）、A 级（7.0-8.9）、B 级（5.0-6.9）、C 级（5.0 以下）
- 优化建议生成 -- 根据评分等级输出核心问题清单、参考爆款标题和修改版本建议

**特色亮点**：

- 标题重合检测：若用户标题与爆款标题重合，直接给予 S 级评分
- 数据驱动评分：所有统计数据必须基于实际召回的爆款数据计算，严禁虚构
- 多种修改风格建议：干货风 / 情绪风 / 猎奇风三种版本
- 反空话规则：所有结论必须有数据支撑

## 4. 使用指南

### 基础使用

**输入格式**：提供 "待评分的标题" 和 "相关主题 / 关键词"。

**输入示例**：
- "帮我看看这个标题'这款防晒霜真的太好用了'怎么样，关键词是防晒"
- "评分'夏天一定要试试这个'，主题是夏日穿搭"
- "'新手必看的护肤技巧' 这个标题如何"

### 完整操作流程

**第一步：理解用户输入并提取关键信息**

识别两个核心要素：
1. 待评分的标题
2. 相关主题 / 关键词（若用户未提供则从标题中提取）

**第二步：判断关键词类型**

- 细分词 / 垂直赛道 -> 直接查询数据
- 具体产品词 -> 直接查询该产品数据
- 泛化词 / 大分类 -> 执行泛化词拓展策略去查询

**第三步：时间范围与数据查询**

- 数据库只包含昨天至 30 天前的数据
- 默认时间范围为最近 30 天
- 日期计算：先获取当前系统时间，再根据用户表达计算 startDate

```bash
python scripts/fetch_xhs_trends.py --keyword <关键词> --start-date <日期>
```

数据不足时如实展示现有数据并提示用户。

**第四步：爆款规律提炼（仅内部执行，不展示给用户）**

所有统计数据必须基于实际召回的爆款数据计算，严禁虚构任何百分比、比例或数值。

规律维度与评分维度一一对应：

1. **主题匹配度规律**：提取实际爆款中踩中的热点关键词及出现频率
2. **结构合规度规律**：统计爆款标题的主流结构占比（数字开头、疑问句、感叹句等）和平均字数
3. **利益清晰度规律**：统计爆款标题中明确标注的具体收益类型
4. **情绪唤醒度规律**：统计爆款标题中的情绪钩子类型（好奇、焦虑、爽点、惊喜等）
5. **稀缺性感知规律**：分析爆款标题中的差异化表述和独家性/限定性表述
6. **合规安全性规律**：检查爆款标题是否存在违规/限流词，统计合规标题占比

**数据计算要求**：
- 所有占比数据必须标注分母（如 "8/10"）
- 必须基于实际召回的爆款标题进行计算
- 若召回数据少于 5 条，需说明 "数据样本较少，规律仅供参考"

**第五步：多维度评分（向用户展示）**

**标题重合判定（在评分前优先检查）**：
- 用户标题与爆款标题完全一致，或仅存在标点符号、空格差异 -> 直接给予 **S 级评分**（9.5 分以上）
- 标注 "该标题与爆款标题重合"，跳过常规评分流程

**总分 10 分，按维度加权计算**：

| 评分维度 | 权重 | 打分标准 |
|---------|------|---------|
| 主题匹配度 | 15% | 10分：踩中当下同赛道热点关键词；<7分：无热点、主题过时 |
| 结构合规度 | 20% | 10分：符合同赛道爆款主流结构（有钩子+有利益）；<5分：平铺直叙无钩子 |
| 利益清晰度 | 25% | 10分：明确标注具体收益/时效/成本；<5分：用户看不到明确价值 |
| 情绪唤醒度 | 20% | 10分：有明确情绪钩子（好奇/焦虑/爽点/惊喜）；<5分：平淡无情绪波动 |
| 稀缺性感知 | 15% | 10分：有差异化/独家性表述；<5分：同质化严重 |
| 合规安全性 | 5% | 10分：无违规/限流词；0分：存在明确违规内容 |

**综合得分** = 主题匹配度 x 0.15 + 结构合规度 x 0.20 + 利益清晰度 x 0.25 + 情绪唤醒度 x 0.20 + 稀缺性感知 x 0.15 + 合规安全性 x 0.05

**输出格式**：

```
## [用户输入的标题]-标题评分报告

**数据说明**：查询到相关小红书爆款标题[X]个，已据此总结相关爆款规律，以下是对您标题的评分。

### 综合得分
**[S/A/B/C]级**-**[X.X]分**（满分10分）

[该等级的具体说明]

### 各维度评分
| 维度 | 得分 | 权重 | 加权得分 |
|------|------|------|----------|
| 主题匹配度 | [X]分 | 15% | [X.X]分 |
| 结构合规度 | [X]分 | 20% | [X.X]分 |
| 利益清晰度 | [X]分 | 25% | [X.X]分 |
| 情绪唤醒度 | [X]分 | 20% | [X.X]分 |
| 稀缺性感知 | [X]分 | 15% | [X.X]分 |
| 合规安全性 | [X]分 | 5% | [X.X]分 |
```

**第六步：优化建议生成（向用户展示）**

**S 级（9.0+ 分）时**：不输出优化建议，纯按爆款规律夸赞。列出 3-5 条同主题的高赞标题，说明相似之处。

**A/B/C 级时**：
1. 核心问题清单 -- 影响最大的问题，每个问题需对应具体的爆款规律依据
2. 参考爆款标题 -- 列出 3-5 条同主题的高赞标题，说明可复用点
3. 修改版本建议 -- 给出 2-3 个修改版本（干货风 / 情绪风 / 猎奇风），标注适用场景和预期效果

**S 级输出格式**：

```

**[若重合：该标题与爆款标题重合，已被市场验证为爆款！]**
```

## 5. 使用场景

1. **创作者发布前验证标题**：在发布笔记前，将备选标题提交评分，根据评分等级和优化建议，选择或修改出更可能成为爆款的标题。
2. **内容运营团队批量评估**：对多个备选标题进行评分，快速筛选出最具爆款潜力的标题，提升团队产出效率。
3. **文案新人学习爆款规律**：通过对比自己写的标题和爆款参考标题，直观理解爆款标题的结构、用词、情绪表达等特征。
4. **A/B 测试前的标题预筛**：在投入流量测试前，先用评分系统预筛出 2-3 个高分标题进行测试，提高测试效率。

## 7. 常见问答

### 安装

**Q: 需要安装什么依赖？**
A: 需要 `requests` 库，如未安装可执行 `pip install requests`。

**Q: 如何配置 API Key？**
A: 请前往 [红狐hub](https://redfox.hk/settings/api-keys?source=github) 获取 API KEY，通过环境变量 `REDFOX_API_KEY` 配置。

### 使用

**Q: 评分维度有哪些？**
A: 6 个维度：主题匹配度（15%）、结构合规度（20%）、利益清晰度（25%）、情绪唤醒度（20%）、稀缺性感知（15%）、合规安全性（5%）。

**Q: 标题与爆款重合会怎样？**
A: 直接给予 S 级评分（9.5 分以上），标注 "该标题已被市场验证为爆款"，跳过常规评分流程。

**Q: 数据范围是多少？**
A: 数据库仅包含昨天至 30 天前的数据，默认时间范围为最近 30 天。

**Q: S 级评分有优化建议吗？**
A: S 级不输出优化建议，转为爆款亮点夸赞和参考标题展示。

### 故障排除

**Q: 召回数据太少怎么办？**
A: 若召回数据少于 5 条，会说明 "数据样本较少，规律仅供参考"。若近 30 天数据仍不足，如实展示现有数据并提示用户。
